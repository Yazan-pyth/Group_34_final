# Методические указания для самостоятельной работы студента по теме «Решение кейса от индустриального партнера: проектирование схемы сбора и первичной очистки данных для автономного датчика. Подготовка к лабораторным работам: изучение форматов данных и методов их эффективной буферизации»

**Дисциплина:** Инженерия встроенных систем машинного обучения  
**Модуль:** 2 (Сбор и предобработка мультимодальных данных для Edge AI)  
**Компетенция:** DL-3 (средний уровень)  
**Объём СРС:** 12 академических часов

## 1. Цель и задачи

**Цель:** Сформировать навыки проектирования систем сбора и первичной обработки данных с сенсоров для автономных Edge AI-устройств, а также освоить эффективные методы буферизации и сериализации данных в условиях ограниченных ресурсов микроконтроллера.

**Задачи:**

- Проанализировать требования к сбору мультимодальных данных для выбранного отраслевого кейса (ТЭК или Транспорт).

- Разработать схему сбора данных с учётом типов сенсоров, частоты дискретизации и доступных интерфейсов (I2C, SPI, UART, I2S).

- Спроектировать пайплайн первичной очистки данных (фильтрация, удаление выбросов, нормализация) на периферийном устройстве.

- Изучить форматы сериализации данных (JSON, Protobuf, MessagePack, CBOR) и методы буферизации (кольцевой буфер, двойная буферизация).

- Подготовиться к выполнению лабораторных работ по калибровке датчиков и передаче данных по BLE/Wi-Fi.

## 2. Этапы выполнения

### Этап 1. Анализ требований к сбору данных (2 ч)

**Задание:**  
Выбрать один из кейсов и определить требования к сбору данных.

**Вариант А. ТЭК — предиктивная диагностика насосного оборудования:**

- Типы сенсоров: акселерометр (вибрация), микрофон (акустика), датчик температуры.

- Частота дискретизации: 1–10 кГц (вибрация), 16–44.1 кГц (акустика), 1 Гц (температура).

- Требования к буферизации: накопление окна данных для ML-инференса (например, 1 секунда данных).

- Ограничения: ограниченный объём RAM (ESP32-S3 — 512 КБ), энергопотребление.

**Вариант Б. Транспорт — детекция препятствий на борту погрузчика:**

- Типы сенсоров: камера (видеопоток), LiDAR/ультразвуковой дальномер, IMU.

- Частота дискретизации: 10–30 FPS (камера), 50–100 Гц (дальномер).

- Требования к латентности: сбор + предобработка \< 20 мс.

- Ограничения: объём данных с камеры требует эффективного сжатия и буферизации.

**Результат этапа:** Документ (1–2 стр.) с перечнем:

- Типов сенсоров и их характеристик.

- Требуемой частоты дискретизации.

- Ожидаемого объёма данных в единицу времени.

- Аппаратных ограничений целевой платформы.

### Этап 2. Проектирование схемы сбора данных (3 ч)

**Задание:**  
Разработать схему подключения и сбора данных для выбранного кейса.

**Основные вопросы для проработки:**

1.  **Выбор интерфейсов подключения:**

    - **I2C:** для датчиков температуры/влажности, акселерометров (BME280, MPU6050) — до 400 кГц.

    - **SPI:** для высокоскоростных датчиков (гироскопы, некоторые акселерометры) — до 80 МГц на ESP32-S3.

    - **UART:** для GPS-модулей, некоторых дальномеров.

    - **I2S:** для цифровых микрофонов (аудиоданные).

2.  **Организация потоков данных:**

    - Использование DMA для автоматической передачи данных из периферийных буферов в память без участия CPU.

    - Настройка прерываний для оповещения о готовности данных.

    - Реализация кольцевого буфера (Ring Buffer) для временного хранения данных.

3.  **Синхронизация мультимодальных данных:**

    - Добавление временных меток к каждому сэмплу (использование аппаратного таймера).

    - Обеспечение корректной синхронизации потоков от разных сенсоров.

**Результат этапа:** Схема подключения датчиков (рисунок) и описание архитектуры сбора данных (2–3 стр.) с указанием:

- Выбранных интерфейсов и пинов подключения.

- Настроек DMA и прерываний.

- Структуры кольцевого буфера.

### Этап 3. Проектирование пайплайна первичной очистки данных (3 ч)

**Задание:**  
Разработать алгоритмы первичной обработки данных на периферийном устройстве.

**Методы очистки данных:**

1.  **Фильтрация шумов:**

    - **Скользящее среднее (Moving Average):** усреднение по окну N отсчётов (N=5–20). Простота реализации, но вносит задержку.

    - **Экспоненциальное сглаживание (EWMA):** Y_new = α \* X + (1-α) \* Y_old, где α — коэффициент сглаживания (0.1–0.5). Меньше задержка, эффективно для медленно меняющихся сигналов.

    - **Медианный фильтр:** удаление импульсных выбросов (замена на медиану окна). Эффективен для защиты от редких артефактов.

    - **Фильтр низких частот (FIR/IIR):** для выделения полезной полосы частот (например, для вибрации — 10 Гц – 1 кГц).

2.  **Обнаружение и удаление выбросов:**

    - Метод Z-оценки: значение считается выбросом, если \|x - μ\| \> k \* σ (k=2–3).

    - Метод межквартильного размаха (IQR): выброс за пределами \[Q1 - 1.5\*IQR, Q3 + 1.5\*IQR\].

3.  **Нормализация и масштабирование:**

    - Минимаксное масштабирование: x_norm = (x - min) / (max - min) для приведения к диапазону \[0, 1\] или \[-1, 1\].

    - Стандартизация: x_norm = (x - μ) / σ для приведения к нулевому среднему и единичной дисперсии.

4.  **Децимация (прореживание):**

    - Снижение частоты дискретизации для уменьшения объёма данных перед передачей.

    - Пример: снижение с 10 кГц до 1 кГц (сохранение каждого 10-го отсчёта или усреднение по блоку).

**Результат этапа:** Описание пайплайна очистки (2–3 стр.) с:

- Выбором методов фильтрации и их обоснованием.

- Алгоритмами обнаружения выбросов.

- Параметрами нормализации и децимации.

- Псевдокодом или блок-схемой алгоритма.

### Этап 4. Изучение форматов данных и методов буферизации (2 ч)

**Задание:**  
Изучить форматы сериализации данных и методы эффективной буферизации для встраиваемых систем.

**Форматы сериализации:**

| Формат               | Размер      | Скорость     | Читаемость       | Поддержка в ESP32     |
|----------------------|-------------|--------------|------------------|-----------------------|
| **JSON**             | Большой     | Медленный    | Человекочитаемый | ArduinoJson, cJSON    |
| **Protobuf**         | Малый       | Быстрый      | Бинарный         | Nanopb                |
| **MessagePack**      | Малый       | Быстрый      | Бинарный         | msgpack-c             |
| **CBOR**             | Малый       | Быстрый      | Бинарный         | tinycbor              |
| **Бинарный (сырой)** | Минимальный | Максимальный | Нечитаемый       | Собственная структура |

**Рекомендации по выбору:**

- **JSON** — для отладки и передачи на сервер (совместимость, человекочитаемость).

- **Protobuf/CBOR** — для production-решений (компактность, скорость).

- **Сырой бинарный** — для межзадачного обмена внутри МК.

**Методы буферизации:**

1.  **Кольцевой буфер (Ring Buffer):**

    - Фиксированный размер, циклическое заполнение.

    - Два указателя: head (запись) и tail (чтение).

    - Преимущества: эффективное использование памяти, отсутствие перемещения данных.

2.  **Двойная буферизация:**

    - Два буфера: один заполняется, другой обрабатывается/передаётся.

    - Переключение по завершении записи/обработки.

    - Преимущества: минимизация задержек, отсутствие конфликтов доступа.

3.  **Связанные списки:**

    - Динамическое выделение блоков под данные.

    - Преимущества: гибкость, недостатки: фрагментация, накладные расходы.

**Результат этапа:** Аналитическая записка (1–2 стр.) с:

- Сравнением форматов сериализации.

- Обоснованием выбора формата для конкретного кейса.

- Описанием выбранного метода буферизации и его реализации.

### Этап 5. Подготовка к лабораторным работам (в рамках времени)

**Задание:**  
Подготовить теоретическую базу и программную среду для выполнения лабораторных работ Модуля 2:

- **ЛР 3:** Калибровка датчиков и управление актуаторами.

- **ЛР 4:** Передача предобработанных данных по BLE/Wi-Fi.

**Рекомендуемые материалы для изучения:**

1.  Работа с I2C/SPI на ESP32-S3 в PlatformIO.

2.  Библиотеки: Wire.h, SPI.h, BLEDevice.h, WiFi.h, PubSubClient.h.

3.  Примеры работы с кольцевым буфером.

4.  Сериализация JSON с ArduinoJson или Protobuf с Nanopb.

**Результат этапа:** Установленные библиотеки, готовые шаблоны проектов, понимание API.

## 3. Структура отчёта

Отчёт должен содержать:

1.  **Титульный лист** (дисциплина, тема, ФИО, группа).

2.  **Техническое задание** — выбранный отраслевой кейс, требования к сбору и обработке данных.

3.  **Схема сбора данных** — подключение датчиков, настройка интерфейсов, DMA и прерываний.

4.  **Пайплайн очистки данных** — методы фильтрации, обнаружение выбросов, нормализация, децимация.

5.  **Сравнение форматов сериализации** — таблица, обоснование выбора.

6.  **Метод буферизации** — описание реализации кольцевого буфера (или двойной буферизации).

7.  **Выводы** — обоснование принятых решений, готовность к лабораторным работам.

8.  **Список источников** — документация, библиотеки, статьи.
