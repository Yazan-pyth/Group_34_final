## КИМ Р2.1. Калибровка датчиков и управление актуаторами по данным с сенсора в реальном времени на ESP32-S3

**Форма проведения:** очная.

**Продолжительность по РПД:** 2 академических часа.

### Цель работы

Освоение методов калибровки аналоговых и цифровых датчиков для компенсации систематических погрешностей, а также разработка системы управления исполнительными устройствами (актуаторами) в реальном времени на основе показаний сенсоров с использованием микроконтроллера ESP32-S3.

### Задача работы

Реализовать калибровку датчиков (температуры, давления или освещенности) с вычислением корректирующих коэффициентов, разработать алгоритм управления сервоприводом или двигателем постоянного тока на основе отфильтрованных показаний датчика в реальном времени. Метриками оптимизации будут являться:

- Точность показаний датчика до и после калибровки (абсолютная и относительная погрешность).

- Время отклика системы управления (задержка от измерения до изменения положения актуатора).

- Стабильность управления в замкнутом контуре (отсутствие автоколебаний/дребезга).

- Вычислительная нагрузка на CPU при выполнении алгоритмов фильтрации и управления.

### Необходимое оборудование и программное обеспечение

- Микроконтроллерная плата ESP32-S3-DevKitC-1 или аналогичная.

- Датчик температуры/влажности (например, DHT22, BME280 или DS18B20) или датчик давления (BMP180/BMP280).

- Эталонный измерительный прибор (цифровой термометр, барометр или мультиметр) для верификации калибровки.

- Актуатор: сервопривод (SG90/MG995) или двигатель постоянного тока с драйвером (L298N/L9110S) и энкодером (опционально).

- Макетная плата, соединительные провода, резисторы (для делителя напряжения при необходимости).

- Персональный компьютер с установленной ОС.

- Среда разработки: PlatformIO (расширение для VSCode) с установленным инструментарием для ESP32-S3.

- Библиотеки: для датчиков (Adafruit Unified Sensor, DHT-sensor-library), для сервопривода (ESP32 Servo Library).

- Утилита для мониторинга последовательного порта и графического отображения данных (Serial Plotter PlatformIO или Python-скрипт с matplotlib).

### Задания для выполнения

#### 1. Подключение датчика и сбор калибровочных данных.

- Подключить выбранный датчик (например, BME280 по I2C) к ESP32-S3.

- Реализовать программу сбора сырых данных (RAW) с датчика.

- Одновременно снимать эталонные показания с помощью внешнего прибора (эталонного термометра/барометра).

- Провести сбор данных в не менее чем 5 различных контрольных точках (например, нагрев/охлаждение датчика, изменение давления).

- Зафиксировать пары значений: "сырое показание датчика" — "эталонное значение" в табличном виде.

- Для аналогового датчика (например, фоторезистор): измерить напряжение на выходе делителя при 5-10 различных уровнях освещенности.

#### 2. Реализация алгоритма калибровки.

#### На основе собранных данных вычислить корректирующие коэффициенты для линейной коррекции

- Методом наименьших квадратов вычислить коэффициенты A и B для уравнения: Y_калибр = A * X_сырое + B.

- В случае нелинейной характеристики (например, для термистора) реализовать кусочно-линейную аппроксимацию (LUT — таблица соответствий) или полиномиальную регрессию второго порядка.

- Реализовать функцию float calibrate(float raw_value), которая применяет вычисленные коэффициенты к сырому значению.

- Сохранить коэффициенты калибровки в энергонезависимой памяти (предпочтительно NVS — Non-Volatile Storage ESP32), чтобы они сохранялись после перезагрузки.

- Реализовать возможность ручной перекалибровки по команде с терминала (UART).

#### 3. Внедрение цифровой фильтрации.

#### Реализовать фильтрацию калиброванных данных для подавления шумов и выбросов

- Простейшее скользящее среднее (Moving Average) с окном N=10-20 отсчетов.

- Медианный фильтр (Median Filter) для удаления импульсных помех.

- Экспоненциальный сглаживающий фильтр (Exponential Moving Average) с коэффициентом α (например, 0.1-0.3).

- Провести сравнительный анализ эффективности каждого фильтра (по критериям: степень сглаживания и вносимая задержка).

- Выбрать оптимальный фильтр для дальнейшего использования в системе управления.

#### 4. Подключение и настройка актуатора.

- Подключить сервопривод к одному из ШИМ-выходов ESP32-S3 (например, GPIO 13) и питание от внешнего источника (через стабилизатор, т.к. ESP32 не может питать сервопривод напрямую).

- Реализовать инициализацию сервопривода с заданием диапазона углов (0°-180°).

- Для двигателя постоянного тока: подключить через драйвер (H-мост), реализовать управление направлением и скоростью вращения (через ШИМ).

- Настроить частоту ШИМ-сигнала: для сервопривода 50 Гц, для двигателя — 1-5 кГц.

#### 5. Реализация замкнутого контура управления (Feedback Control).

- Разработать алгоритм управления актуатором на основе калиброванных и отфильтрованных данных с датчика.

#### Примеры сценариев

- Термостатирование: Поддержание заданной температуры. Если измеренная температура ниже уставки — включить нагреватель или повернуть сервопривод для открытия заслонки; если выше — выключить или прикрыть. Реализовать управление с гистерезисом для предотвращения частых переключений.

- Слежение за уровнем освещенности: Если освещенность падает ниже порога — повернуть сервопривод в положение, открывающее шторы или включающее светодиод с плавным изменением яркости.

- Реализовать усложненный регулятор (ПИД-регулятор) с расчетом коэффициентов Kp, Ki, Kd (предварительно подобрать опытным путем или методом Циглера-Николса).

- Организовать вывод в монитор порта графика изменения измеряемой величины, уставки и управляющего сигнала для визуальной оценки качества регулирования.

#### 6. Измерение и оценка показателей.

- Замерить латентность цикла управления (время от момента взятия отсчета до выдачи управляющего сигнала на ШИМ).

- Оценить точность поддержания уставки (среднеквадратичное отклонение, максимальная ошибка).

- Проанализировать, насколько калибровка улучшила точность по сравнению с некалиброванными показаниями.

- Провести стресс-тест на длительность 15-20 минут с фиксацией всех параметров (для оценки стабильности регулятора).

#### Проверка корректности (Тест целостности данных)

- Верификация фильтрации: Подача импульсной помехи (кратковременное изменение показаний) должна быть подавлена без выброса управления.

- Верификация управления: При ступенчатом изменении уставки переходный процесс должен быть апериодическим или с затухающими колебаниями (без расходящихся автоколебаний).

- Тест сохранения калибровки: После перезагрузки платы коэффициенты должны восстанавливаться из NVS.

#### Критерии успешной проверки

- Погрешность калиброванного датчика не превышает ±1.5% от полного диапазона.

- Время отклика системы управления менее 200 мс.

- Отсутствие перерегулирования более 20% при ступенчатом воздействии.

- Стабильность коэффициентов калибровки после перезагрузки (проверка через чтение NVS).

#### Чеклист для production-развертывания

- Коэффициенты калибровки защищены от случайного изменения (CRC-контроль в NVS).

- Фильтр не вносит фазового запаздывания более 3-х периодов дискретизации.

- ПИД-регулятор имеет защиту от интегрального насыщения (anti-windup).

- Управляющий сигнал ограничен по максимуму/минимуму во избежание повреждения актуатора.

- Реализована функция автоматического аварийного отключения актуатора при потере связи с датчиком (таймаут чтения > 2 секунд).

- Проведен тест работы устройства при пониженном напряжении питания.

- Код успешно проходит статический анализ (отсутствие потенциальных делений на ноль и переполнений переменных).

## КИМ Р2.2. Передача предобработанных данных по BLE/Wi-Fi с акцентом на IoT-кейсы ТЭК/Транспорт

**Форма проведения:** очная.

**Продолжительность по РПД:** 2 академических часа.

### Цель работы

Освоение методов беспроводной передачи предварительно обработанных телеметрических данных с использованием протоколов BLE (Bluetooth Low Energy) и Wi-Fi в контексте отраслевых задач топливно-энергетического комплекса (ТЭК) и транспорта. Особое внимание уделяется обеспечению достоверности, защите и энергоэффективности передачи при работе в условиях ограниченных ресурсов микроконтроллера.

### Задача работы

Реализовать на базе ESP32-S3 систему сбора данных с датчиков (температуры, давления, вибрации), выполнить их первичную обработку (фильтрация, децимация, вычисление статистик), и организовать передачу предобработанных данных в формате JSON/Protobuf через BLE (для ближней связи) и Wi-Fi (для передачи на удаленный сервер MQTT/HTTP). Метриками оптимизации будут являться:

- Объем передаваемых данных в единицу времени (байт/с).

- Время передачи пакета (латентность) для BLE и Wi-Fi каналов.

- Потребляемый ток при передаче по каждому из протоколов.

- Достоверность передачи (процент потерянных/искаженных пакетов).

- Степень сжатия данных за счет предобработки (децимации и агрегации).

### Необходимое оборудование и программное обеспечение

- Микроконтроллерная плата ESP32-S3-DevKitC-1 или аналогичная (с поддержкой BLE и Wi-Fi).

- Датчики: акселерометр/гироскоп (например, MPU6050/MPU9250 по I2C), датчик температуры/давления (BME280), вибродатчик (пьезоэлектрический через АЦП).

- Смартфон или второй ESP32 в качестве BLE-приемника (клиент) с установленным приложением для отображения данных (например, nRF Connect / BLE Scanner).

- Wi-Fi точка доступа с доступом к сети Интернет (для подключения к MQTT-брокеру).

- Облачная платформа: MQTT-брокер (например, HiveMQ Cloud, Mosquitto) или HTTP-сервер (например, Flask на локальной машине для приема данных).

- Персональный компьютер с установленной ОС.

- Среда разработки: PlatformIO (расширение для VSCode) с инструментарием для ESP32-S3.

- Утилиты для мониторинга последовательного порта, Wireshark или анализатор Bluetooth (опционально).

- Библиотеки: WiFi.h, BluetoothSerial.h, BLEDevice.h (ESP32 BLE Arduino), PubSubClient.h (для MQTT), ArduinoJson.h (для работы с JSON), или Nanopb (для Protobuf).

### Задания для выполнения

#### 1. Настройка сбора данных и предобработка на устройстве.

- Подключить датчики (акселерометр/гироскоп и датчик давления/температуры) к ESP32-S3 по интерфейсу I2C.

- Настроить частоту опроса датчиков: для акселерометра — 100 Гц (вибродиагностика), для датчика температуры/давления — 1 Гц (технологические параметры).

#### Реализовать первичную обработку (Edge Computing) непосредственно на МК перед передачей

- Децимация: Снижение частоты дискретизации акселерометра с 100 Гц до 10 Гц (сохранение каждого 10-го отсчета или усреднение по окну).

- Расчет статистик: Вычисление скользящего среднего, дисперсии, RMS (среднеквадратичного значения) и пиковых значений вибрации по окну из 50 отсчетов.

- Обнаружение событий: Реализация детектора превышения порога вибрации, при котором формируется "тревожный" пакет с повышенным приоритетом.

#### Сформировать пакет данных, содержащий

- Метку времени (timestamp от системного таймера).

- Идентификатор узла (Node ID).

- Калиброванные и отфильтрованные значения параметров.

- CRC-контрольную сумму или хеш (например, CRC32) для проверки целостности.

- Выполнить сериализацию пакета в JSON-строку (человекочитаемый формат) и в бинарный формат Protobuf (компактный и производительный).

#### 2. Реализация BLE-передачи (ближняя связь).

- Настроить ESP32-S3 в режиме BLE-периферийного устройства (Peripheral).

- Создать сервис (Service) с пользовательским UUID.

- Создать характеристику (Characteristic) с правами Read/Notify для передачи телеметрических данных.

#### Реализовать отправку предобработанных данных

- Каждые 5 секунд отправлять статистический пакет (усредненные значения).

- При обнаружении "тревожного" события (превышение вибрации) отправлять мгновенное уведомление (Notify) с высокоприоритетными данными.

- Настроить скорость передачи (MTU-размер) для оптимизации пропускной способности.

- Провести прием данных на смартфоне через приложение nRF Connect для визуальной верификации.

#### 3. Реализация Wi-Fi/MQTT-передачи (удаленная связь).

- Настроить ESP32-S3 в режиме клиента Wi-Fi (STA) для подключения к точке доступа.

#### Настроить подключение к MQTT-брокеру

- Указать адрес брокера, порт (1883 или 8883 для TLS), логин/пароль (при необходимости).

- Реализовать механизм переподключения при разрыве связи (retry с экспоненциальной задержкой).

#### Организовать публикацию данных в топики

- devices/node_XYZ/sensors — для регулярной телеметрии (частота 1 раз в 10 секунд).

- devices/node_XYZ/alerts — для тревожных сообщений.

- Настроить QoS (Quality of Service) на уровне 1 (гарантированная доставка) для тревожных пакетов.

- Реализовать получение команд с брокера (подписка на топик devices/node_XYZ/commands) для удаленного изменения параметров опроса (частота, пороги) и перезагрузки устройства.

#### 4. Сравнительный анализ протоколов и оптимизация энергопотребления.

#### Провести сравнительные замеры метрик передачи

- Для Wi-Fi: время установления соединения, время передачи, ток в режиме передачи.

#### Реализовать интеллектуальную стратегию маршрутизации данных

- По умолчанию данные передаются через BLE (экономичный режим).

- При потере BLE-соединения или при наступлении критического события автоматически переключаться на Wi-Fi.

- Реализовать "пакетный" режим (store-and-forward): при отсутствии сети данные сохраняются во Flash/NVS и отправляются после восстановления связи (Critical для транспорта и ТЭК).
