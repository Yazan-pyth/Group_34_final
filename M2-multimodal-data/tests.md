## Л2.1. Сенсоры, потоки данных, буферизация и предобработка

### 1. Для непрерывной обработки потокового аудио без пропусков при постоянном поступлении новых сэмплов применяют:

- **А)** Отдельный файл на каждый сэмпл
- **Б)** Одиночную переменную
- **В)** Хранение всего потока в одном массиве до конца сессии
- **Г)** Кольцевой (ring) буфер

### 2. Модель обучалась на изображениях, нормированных в [-1, 1], а на устройстве вход подаётся в [0, 255]. Что произойдёт?

- **А)** Модель ускорится за счёт большего диапазона
- **Б)** Уменьшится footprint модели
- **В)** Ничего, нормализация не влияет на инференс
- **Г)** Точность резко упадёт из-за сдвига распределения входов

### 3. Частота дискретизации сигнала критична, потому что она:

- **А)** Определяет число классов модели
- **Б)** Влияет только на размер корпуса устройства
- **В)** Задаёт, насколько корректно восстанавливается сигнал во времени (теорема Котельникова)
- **Г)** Управляет работой DMA

### 4. Для аудио-задачи (wake word) типичным входом модели служит:

- **А)** Спектрограмма или MFCC-признаки над окном сигнала
- **Б)** Карта глубины сцены
- **В)** Таблица весов сети
- **Г)** Сырые RGB-пиксели

### 5. Почему предобработка на устройстве обязана в точности повторять предобработку при обучении?

- **А)** Иначе уменьшится доступный Flash
- **Б)** Иначе распределение входов разойдётся с обучающим и качество упадёт
- **В)** Иначе отключится DMA-контроллер
- **Г)** Иначе модель сменит архитектуру

### 6. IMU в edge-устройстве измеряет:

- **А)** Температуру окружающей среды
- **Б)** Уровень звука
- **В)** Ускорение и угловую скорость
- **Г)** Освещённость сцены

### 7. Кольцевой буфер переполняется, потому что задача-потребитель не успевает разгребать данные. Что произойдёт при неверной обработке?

- **А)** Модель автоматически ускорится
- **Б)** Сенсор изменит тип выходных данных
- **В)** Часть данных будет перезаписана и потеряна
- **Г)** Увеличится тактовая частота CPU

### 8. Зачем нужна временная синхронизация нескольких сенсоров (камера + IMU)?

- **А)** Чтобы корректно сопоставлять события во времени при слиянии данных
- **Б)** Чтобы увеличить число GPIO
- **В)** Чтобы уменьшить число классов
- **Г)** Чтобы отключить лишнюю периферию

### 9. Когда предобработку выгодно выполнять прямо на MCU, а не передавать сырые данные дальше?

- **А)** Всегда, независимо от ресурсов
- **Б)** Когда она проста и заметно сокращает объём передаваемых данных
- **В)** Когда она требует GPU
- **Г)** Только если есть Wi-Fi

### 10. Что такое «окно» в обработке сигнала?

- **А)** Графический интерфейс ОС
- **Б)** Буфер для хранения весов
- **В)** Фрагмент последовательности фиксированной длины, подаваемый на вход модели
- **Г)** Список выходных классов

## Л2.2. Синтетические данные, калибровка и облегчённая предобработка

### 1. Для чего служит калибровочный датасет при post-training INT8-квантовании?

- **А)** Для увеличения числа слоёв
- **Б)** Для дообучения весов модели
- **В)** Для настройки скорости UART
- **Г)** Для оценки диапазонов активаций и подбора scale/zero-point

### 2. После INT8-квантования точность упала на 15%. Наиболее вероятная причина, связанная с калибровкой:

- **А)** Слишком короткое имя модели
- **Б)** Избыток классов в датасете
- **В)** Слишком высокая частота CPU
- **Г)** Нерепрезентативный калибровочный набор из однотипных «идеальных» примеров

### 3. Каким должен быть калибровочный набор для корректного квантования?

- **А)** Максимально случайным шумом
- **Б)** Состоящим только из выбросов и аварийных режимов
- **В)** Репрезентативным для реальных рабочих входов
- **Г)** Как можно меньшим по объёму

### 4. Синтетические данные в промышленном кейсе полезны прежде всего тем, что:

- **А)** Полностью заменяют реальные данные и испытания
- **Б)** Гарантируют 100% точность модели
- **В)** Увеличивают частоту NPU
- **Г)** Позволяют покрыть редкие и опасные (аварийные) сценарии

### 5. Главный риск использования синтетических данных — это:

- **А)** Domain gap: расхождение с реальным распределением
- **Б)** Слишком низкая скорость UART
- **В)** Невозможность хранения на диске
- **Г)** Автоматическое переобучение модели

### 6. Как корректно применять синтетические данные в обучении?

- **А)** Использовать только их, без реальных
- **Б)** Комбинировать с реальными и валидировать отдельно на реальном наборе
- **В)** Не проверять итоговое качество
- **Г)** Заменять ими тестовую выборку

### 7. На устройстве использован другой алгоритм resize, чем при обучении. Последствие:

- **А)** Никаких — resize не влияет на инференс
- **Б)** Компилятор не соберёт прошивку
- **В)** Систематический сдвиг входов и падение качества
- **Г)** Уменьшение объёма Flash

### 8. После упрощения предобработки под ограничения MCU что необходимо проверить?

- **А)** Качество модели и скорость на целевом устройстве совместно
- **Б)** Только число операций в коде
- **В)** Только размер бинарного файла
- **Г)** Только имя выходного класса

### 9. Почему нормализация входа важна для корректного инференса?

- **А)** Она уменьшает число классов
- **Б)** Она приводит входы к распределению, ожидаемому моделью
- **В)** Она управляет контроллером DMA
- **Г)** Она сжимает веса

### 10. Что необходимо сравнить сразу после калибровки и квантования?

- **А)** FP32- и INT8-версии на одном и том же наборе
- **Б)** Две принципиально разные задачи
- **В)** Только размеры файлов моделей
- **Г)** Скорость обучения до и после
