## КИМ Р3.1. Интеграция задачи сбора данных и задачи ML-инференса с приоритезацией потоков на ESP32-S3

**Форма проведения:** очная.

**Продолжительность по РПД:** 2 академических часа.

### Цель работы

Освоение методов организации многозадачности в реальном времени на базе FreeRTOS для ESP32-S3, включая приоритезацию потоков, синхронизацию и обмен данными между задачей сбора данных (высокочастотный опрос датчиков) и задачей ML-инференса (ресурсоемкая обработка). Особое внимание уделяется обеспечению детерминизма сбора данных и минимизации задержек выполнения критических задач.

### Задача работы

Спроектировать и реализовать многопоточную систему на ESP32-S3, в которой:

- Высокоприоритетная задача A: Выполняет сбор данных с датчиков (акселерометр/гироскоп) с частотой 100 Гц с использованием аппаратных прерываний и DMA, накапливает данные в кольцевом буфере.

- Низкоприоритетная задача B: Выполняет ML-инференс (распознавание жестов/вибрационных паттернов) на накопленных данных с использованием предварительно загруженной модели TensorFlow Lite Micro, не блокируя задачу сбора.

- Задача мониторинга C: Периодически выводит в UART информацию о состоянии системы (свободная память, загрузка CPU, количество обработанных кадров). Метриками оптимизации будут являться:

- Задержка между появлением нового отсчета и его попаданием в буфер (максимальное время).

- Частота выполнения ML-инференса (кадров в секунду).

- Стабильность частоты сбора данных (отклонение от 100 Гц).

- Наличие/отсутствие потерь данных (Overflow) и ресурсных конфликтов.

- Загрузка процессора (CPU Load) в процентах.

### Необходимое оборудование и программное обеспечение

- Микроконтроллерная плата ESP32-S3-DevKitC-1 или аналогичная (с поддержкой двухъядерного режима).

- Датчик движения/вибрации (акселерометр ADXL345 или MPU6050 по I2C/SPI) для генерации потоковых данных.

- Персональный компьютер с установленной ОС.

- Среда разработки: PlatformIO (расширение для VSCode) с инструментарием для ESP32-S3.

#### Библиотеки

- TensorFlow Lite Micro (ядро для выполнения инференса на ESP32-S3).

- esp_timer или esp_freertos_hooks для высокоточных замеров времени.

- freertos/FreeRTOS.h, freertos/task.h, freertos/queue.h, freertos/semphr.h.

- Предварительно обученная и квантованная модель TinyML (например, модель распознавания жестов на основе 3-осевого акселерометра, сконвертированная в C-массив через xxd или TensorFlow Lite Converter).

- Утилита для мониторинга последовательного порта (Serial Monitor PlatformIO).

- Логический анализатор или осциллограф для верификации таймингов прерываний (опционально).

### Задания для выполнения

#### 1. Подготовка ML-модели и интеграция в проект.

- Загрузить предварительно обученную модель Keras/TFLite (например, модель распознавания жестов на основе данных с 3-осевого акселерометра с частотой 50-100 Гц).

- Выполнить пост-тренировочное квантование модели (float32 → int8) с помощью TensorFlow Lite Converter.

- Сконвертировать квантованную модель в C-массив (с помощью xxd или Python-скрипта) для встраивания в прошивку.

#### Интегрировать массив в проект и инициализировать интерпретатор TensorFlow Lite Micro

- Выделить память для тензоров (входной/выходной).

- Загрузить модель в интерпретатор.

- Проверить успешность выполнения эталонного инференса на тестовых данных в режиме эмуляции (на ПК).

#### 2. Проектирование архитектуры многозадачной системы.

#### Спроектировать взаимодействие между задачами с использованием механизмов FreeRTOS

- Кольцевой буфер (Ring Buffer): Реализовать потокобезопасный кольцевой буфер для хранения сырых данных с датчика (с использованием семафоров или мьютексов). Размер буфера должен вмещать не менее 512 отсчетов (для накопления кадра для ML-инференса).

- Очереди сообщений: Создать очередь для передачи уведомлений о готовности нового набора данных от задачи сбора к задаче инференса.

#### Создать три задачи FreeRTOS с различными приоритетами

- Task_DataCapture (приоритет: высокий, например, 5) — сбор данных с датчика.

- Task_Inference (приоритет: низкий, например, 2) — выполнение ML-инференса.

- Task_Monitor (приоритет: низший, например, 1) — вывод системной информации в UART каждые 2 секунды.

- Назначить задачи на разные ядра процессора (для ESP32-S3): задачу сбора данных — на ядро 0 (ядро с поддержкой прерываний), задачу инференса — на ядро 1.

#### 3. Реализация высокоприоритетной задачи сбора данных.

- Настроить аппаратный таймер или таймер группы для генерации прерываний с частотой 100 Гц (период 10 мс).

- В обработчике прерывания (ISR) — выполнять только минимально необходимые действия: чтение данных из аппаратного регистра датчика (по I2C/SPI) и помещение в кольцевой буфер.

- Использовать механизм xSemaphoreGiveFromISR() для оповещения задачи Task_DataCapture о готовности нового блока данных.

#### В теле задачи Task_DataCapture

- Ожидать семафор (приоритетное прерывание).

- Извлекать данные из аппаратного буфера (DMA) и записывать их в кольцевой буфер.

- При накоплении 50-100 отсчетов (полный кадр для инференса) — отправлять уведомление в очередь для задачи инференса.

- Добавить защиту критических секций при работе с общим кольцевым буфером.

#### 4. Реализация низкоприоритетной задачи ML-инференса.

#### В теле задачи Task_Inference

- Ожидать сообщение из очереди (уведомление о готовности кадра данных).

- Скопировать данные из кольцевого буфера во входной тензор модели (с учетом квант-параметров).

- Выполнить инференс с помощью tflite->Invoke().

- Считать результаты из выходного тензора.

- Сформировать и вывести в UART результат (класс жеста/вероятность) с временной меткой.

- Если инференс выполняется дольше интервала сбора данных, задача может быть вытеснена более приоритетной задачей сбора — это допустимо.

- Использовать мьютекс для защиты доступа к кольцевому буферу во время чтения данных для инференса.

#### 5. Реализация задачи мониторинга и системной отладки.

#### Создать задачу Task_Monitor, которая каждые 2 секунды выводит в UART

- % загрузки CPU (с использованием esp_get_free_heap_size() или uxTaskGetSystemState()).

- Количество свободной динамической памяти (heap).

- Количество выполненных инференсов.

- Количество потерянных отсчетов (переполнение буфера, если было).

- Тактовые метки для оценки задержек.

#### Реализовать команды управления через UART

- pause — приостановить задачу инференса.

- resume — возобновить.

- stats — вывести расширенную статистику.

#### 6. Измерение и анализ производительности.

#### Провести серию замеров

- Максимальная задержка сбора (ISR→буфер): Измерить с помощью высокоточного таймера (esp_timer_get_time()) и вывести максимальное отклонение.

- Пропускная способность: Количество кадров инференса в секунду.

- Потери данных: Количество случаев переполнения кольцевого буфера за 5 минут работы.

- CPU Load: Загрузка каждого ядра при помощи esp_cpu_get_load().

- Построить сравнительную таблицу для различных конфигураций приоритетов (для нахождения оптимальной).

- Проверить детерминизм системы при внешних прерываниях (например, отправка пакета по UART с высоким приоритетом).

## КИМ Р3.2. Настройка OTA-обновлений и логирование метрик инференса

**Форма проведения:** очная.

**Продолжительность по РПД:** 2 академических часа.

### Цель работы

Освоение методов организации беспроводного обновления прошивки (OTA — Over-The-Air) для микроконтроллеров семейства ESP32-S3, а также разработка системы логирования ключевых метрик работы ML-модели (точность, задержка, ресурсы) с сохранением во внутренней Flash-памяти и передачей на удаленный сервер для последующего анализа.

### Задача работы

Реализовать на базе ESP32-S3 систему, выполняющую периодический ML-инференс, логирующую метрики его работы (время выполнения, результат, уверенность модели, загрузка памяти) в энергонезависимую память (SPIFFS/LittleFS), и поддерживающую удаленное обновление прошивки через Wi-Fi по протоколу HTTP/HTTPS с защитой от сбоев (двухбанковая схема или механизм отката). Метриками оптимизации будут являться:

- Время выполнения OTA-обновления (сек).

- Объем используемой Flash-памяти для хранения логов (байт).

- Надежность OTA-процесса (успешность обновления при сбоях питания/связи).

- Скорость записи/чтения логов в файловую систему.

- Точность логируемых временных меток (мс).

### Необходимое оборудование и программное обеспечение

- Микроконтроллерная плата ESP32-S3-DevKitC-1 или аналогичная (с достаточным объемом Flash — не менее 4 МБ).

- Датчики (акселерометр/гироскоп, датчик температуры/влажности) для генерации потока данных.

#### Определить перечень логируемых метрик для каждого инференса

- timestamp — временная метка (Unix-время или мс от старта).

- inference_time_ms — время выполнения инференса (мс).

- free_heap — свободная динамическая память до инференса (байт).

#### Реализовать процесс загрузки и установки новой прошивки
