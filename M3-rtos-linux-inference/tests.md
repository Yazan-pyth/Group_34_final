## Л3.1. FreeRTOS и детерминированный ML-инференс на MCU

### 1. Как правильно организовать одновременный сбор данных с сенсора и тяжёлый ML-инференс в FreeRTOS?

- **А)** Выполнять оба внутри одной ISR
- **Б)** Запускать обе операции в случайном порядке в одном цикле
- **В)** Вынести в отдельные задачи и связать очередью
- **Г)** Отключить планировщик на время инференса

### 2. Низкоприоритетная задача захватила мьютекс, нужный высокоприоритетной, и та простаивает. Это называется:

- **А)** Context switch
- **Б)** Deadlock
- **В)** Race condition
- **Г)** Priority inversion (инверсия приоритетов)

### 3. Для системы реального времени критично оценивать прежде всего:

- **А)** Среднее время выполнения инференса
- **Б)** Минимальное время одного запуска
- **В)** Worst-case latency (худший допустимый случай)
- **Г)** Размер файла README

### 4. Две задачи бесконечно ждут ресурсы, захваченные друг другом. Это:

- **А)** Priority inversion
- **Б)** Ошибка квантования
- **В)** Watchdog reset
- **Г)** Deadlock (взаимная блокировка)

### 5. Долгий синхронный инференс в высокоприоритетной задаче опасен тем, что:

- **А)** Уменьшает объём Flash
- **Б)** Повышает точность модели
- **В)** Всегда вызывает brownout
- **Г)** Задерживает обслуживание других критичных задач и срывает дедлайны

### 6. Что нельзя делать внутри обработчика прерывания (ISR) в FreeRTOS?

- **А)** Долгие блокирующие операции и ожидание мьютекса
- **Б)** Коротко зафиксировать флаг события
- **В)** Отправить сигнал задаче через ISR-safe API
- **Г)** Считать один регистр

### 7. Для безопасной передачи данных между задачей-сборщиком и задачей-инференсом используют:

- **А)** Глобальную переменную без защиты
- **Б)** Очередь (queue)
- **В)** Прямой вызов функции инференса из ISR
- **Г)** Общий буфер без синхронизации

### 8. Что такое race condition?

- **А)** Плановый перезапуск сервиса
- **Б)** Снижение точности после квантования
- **В)** Ошибка из-за несогласованного одновременного доступа к общим данным
- **Г)** Сбой интерфейса UART

### 9. Зачем в системе нужен сторожевой таймер (watchdog)?

- **А)** Для фильтрации входного сигнала
- **Б)** Для квантования модели
- **В)** Для обучения на устройстве
- **Г)** Для автоматического перезапуска при зависании

### 10. Отладочное логирование в real-time системе следует ограничивать, потому что оно:

- **А)** Влияет на временные характеристики и может срывать дедлайны
- **Б)** Навсегда отключает прерывания
- **В)** Уменьшает число задач
- **Г)** Снижает точность модели

## Л3.2. Embedded Linux на Rockchip и базовый деплой ML-сервисов

### 1. После обновления системы RKNN-модель перестала запускаться с ошибкой инициализации. Наиболее вероятная причина:

- **А)** На плате отключён Wi-Fi
- **Б)** Слишком длинное имя файла модели
- **В)** Модель обучалась в PyTorch, а не в TensorFlow
- **Г)** Несовместимость версий RKNN-Toolkit (конвертация) и runtime/драйвера на устройстве

### 2. Для управления автозапуском ML-сервиса и его перезапуском при сбое в Linux используют:

- **А)** Компилятор gcc
- **Б)** Драйвер камеры
- **В)** RKNN-Toolkit2
- **Г)** systemd (unit-сервис)

### 3. Что необходимо предусмотреть для безопасного OTA-обновления edge-устройства в поле?

- **А)** Только наличие Wi-Fi
- **Б)** Только Docker-контейнер
- **В)** Версионирование, проверку целостности и возможность отката (rollback)
- **Г)** Увеличенный объём RAM

### 4. Для чего контейнеризация (Docker) может быть полезна на edge, но не является центральной темой?

- **А)** Она ускоряет NPU в любом случае
- **Б)** Она уменьшает размер модели
- **В)** Она автоматически квантует веса
- **Г)** Она изолирует зависимости и повышает воспроизводимость деплоя

### 5. Почему Docker не решает ключевых задач Edge AI сам по себе?

- **А)** Он не работает на архитектуре ARM
- **Б)** Он не устраняет проблем конвертации модели и аппаратного ускорения
- **В)** Он запрещён в embedded Linux
- **Г)** Он несовместим с systemd

### 6. Что такое device node (/dev/...) в Linux?

- **А)** Файл-интерфейс для доступа к устройству/периферии
- **Б)** Папка с обучающим датасетом
- **В)** Параметр квантования модели
- **Г)** Раздел swap-памяти

### 7. При запуске ML-сервиса на Rockchip обязательно контролировать температуру SoC, потому что:

- **А)** Температура меняет число классов модели
- **Б)** Перегрев вызывает throttling и нестабильность инференса
- **В)** Температура влияет только на Wi-Fi
- **Г)** Иначе не запустится systemd

### 8. Что означает «воспроизводимый деплой»?

- **А)** Одинаковая процедура даёт одинаковый результат на всех целевых устройствах
- **Б)** Каждый запуск отличается от предыдущего
- **В)** Приложение работает без зависимостей
- **Г)** Модель дообучается при каждом старте

### 9. Как правильно организовать логирование на edge-устройстве с ограниченной памятью?

- **А)** Логировать все входные данные без ограничений
- **Б)** Логировать только время загрузки ОС
- **В)** Логировать ключевые метрики и ошибки с ротацией и ограничением объёма
- **Г)** Отключить логирование полностью

### 10. Что необходимо измерять при эксплуатации ML-сервиса в продуктиве?

- **А)** Только uptime устройства
- **Б)** Latency, загрузку CPU/NPU, память и стабильность
- **В)** Только размер образа системы
- **Г)** Только версию ядра Linux
