# Методические указания для самостоятельной работы студента по теме «Решение кейса от индустриального партнера: анализ ограничений целевой платформы ТЭК/Транспорт и проектирование архитектуры на C++. Подготовка к лабораторным работам: изучение документации к MCU/MPU и принципов работы NPU RKNN»

**Дисциплина:** Инженерия встроенных систем машинного обучения  
**Модуль:** 1 (Архитектура embedded-платформ и программирование периферии на C/C++)  
**Компетенция:** PL-4 (продвинутый уровень)  
**Объём СРС:** 10 академических часов

## 1. Цель и задачи

**Цель:** Сформировать навыки анализа аппаратных ограничений целевых платформ (ESP32-S3, STM32, Rockchip RK3588/RK3576) и проектирования эффективной архитектуры встроенного ПО на C++ для Edge AI-решений в отраслях ТЭК и Транспорт.

**Задачи:**

- Проанализировать требования к Edge AI-устройству для выбранной отрасли.

- Изучить архитектурные ограничения выбранной платформы (память, тактовая частота, энергопотребление, наличие NPU).

- Разработать модульную архитектуру ПО на C++ с учётом этих ограничений.

- Освоить документацию к ESP-IDF, STM32Cube, RKNN-Toolkit2.

- Подготовиться к выполнению лабораторных работ.

## 2. Этапы выполнения

### Этап 1. Выбор кейса и формулирование ТЗ (2 ч)

**Выбрать один из двух кейсов:**

| Отрасль       | Задача                                                          | Ключевые требования                                                                                    |
|---------------|-----------------------------------------------------------------|--------------------------------------------------------------------------------------------------------|
| **ТЭК**       | Автономный датчик предиктивной диагностики насоса               | Сбор вибрации/акустики, TinyML-детекция аномалий на ESP32, передача по MQTT, работа от батареи \>2 лет |
| **Транспорт** | Edge AI-модуль детекции препятствий для беспилотного погрузчика | Работа на RK3588 с NPU, RKNN, интеграция с FreeRTOS, латентность \<50 мс                               |

**Результат:** Техническое задание (1–2 стр.) с функциональными требованиями, требованиями к производительности, энергопотреблению и надёжности.

### Этап 2. Анализ ограничений платформы (3 ч)

Изучить документацию и заполнить таблицу ключевых параметров:

| Параметр           | ESP32-S3                  | STM32 (L4/F4) | RK3588          |
|--------------------|---------------------------|---------------|-----------------|
| Тактовая частота   | до 240 МГц (двухъядерный) | до 180 МГц    | 4×A76 + 4×A55   |
| RAM                | 512 КБ                    | до 512 КБ     | до 32 ГБ        |
| Flash              | до 16 МБ (внешняя)        | до 2 МБ       | –               |
| NPU                | –                         | –             | до 6 TOPS, INT8 |
| Беспроводная связь | Wi-Fi + BLE               | опционально   | опционально     |
| Инструменты        | ESP-IDF, TFLite Micro     | STM32Cube.AI  | RKNN-Toolkit2   |

**Результат:** Таблица с выявленными ограничениями и их влиянием на архитектуру.

### Этап 3. Изучение принципов NPU Rockchip и RKNN (2 ч)

Самостоятельно изучить:

- Архитектуру RKNPU (матричные умножители, поддержка INT8).

- RKNN Software Stack (Toolkit2, Runtime, драйвер).

- Процесс конвертации ONNX/TensorFlow → RKNN с калибровкой и квантованием.

- C/C++ API для выполнения инференса на устройстве.

**Результат:** Конспект (2–3 стр.) с описанием основных шагов конвертации и вызова модели.

### Этап 4. Проектирование архитектуры на C++ (3 ч)

Разработать модульную архитектуру встроенного ПО:

- **Модули:** инициализация периферии, сбор данных (DMA/прерывания), предобработка, ML-инференс (TFLite Micro / RKNN), передача данных, управление питанием (режимы сна).

- **Управление ресурсами:** статическое выделение памяти, избегание динамической аллокации.

- **Многозадачность:** для ESP32-S3 – FreeRTOS; для STM32 – HAL + CMSIS.

- **Энергосбережение:** переход в Light/Deep Sleep, динамическое управление частотой.

**Результат:** Архитектурная схема и описание модулей (3–5 стр.) с указанием интерфейсов и потоков данных.

### Этап 5. Подготовка к лабораторным работам (в рамках времени)

Установить и настроить среду разработки:

- PlatformIO / ESP-IDF для ESP32-S3.

- STM32CubeMX + STM32Cube.AI для STM32.

- RKNN-Toolkit2 + примеры из rknn_model_zoo.

## 3. Структура отчёта

Отчёт должен содержать:

1.  **Титульный лист** (дисциплина, тема, ФИО, группа).

2.  **Техническое задание** (описание кейса, требования).

3.  **Анализ ограничений платформы** (таблица, выводы).

4.  **Конспект по RKNN** (краткое описание архитектуры и процесса конвертации).

5.  **Архитектура ПО** (диаграмма компонентов, описание модулей, схема потоков данных).

6.  **Выводы** (обоснование решений, готовность к ЛР).

7.  **Список источников** (документация, руководства).
