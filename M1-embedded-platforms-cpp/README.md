# Модуль 1. Архитектура embedded-платформ и программирование периферии на C/C++

## Назначение модуля

Модуль входит в рабочую программу дисциплины «Инженерия встроенных систем машинного обучения» и обеспечивает освоение индикаторов `PL-4.2`.

**Трудоёмкость модуля:** 18 академических часов.

## Самостоятельная работа

Полные методические указания: [самостоятельная работа модуля](independent-work.md).

**Объём СРС:** 10 академических часов.  
**Проверяемые компетенции:** `PL-4`.

## КИМ Р1.1. Аппаратно-программная разработка на MCU: настройка среды (PlatformIO/ESP-IDF), программирование чтения данных с сенсоров и управление GPIO на C++ (на базе ESP32-S3)

**Форма проведения:** очная.

**Продолжительность по РПД:** 3 академических часа.

### Цель работы

Освоение базовых принципов разработки встроенных систем на базе микроконтроллера ESP32-S3, включая настройку профессиональной среды разработки (PlatformIO или ESP-IDF), реализацию драйверов для чтения данных с цифровых и аналоговых сенсоров, а также управление периферийными устройствами через GPIO с использованием языка C++.

### Задача работы

Настроить инструментальную среду разработки, реализовать проект, выполняющий циклический опрос датчиков (температуры/влажности и освещенности) и управляющий внешним светодиодом на основе пороговых значений. Метриками успешности выполнения будут являться:

- Стабильность работы проекта (отсутствие сбоев и перезагрузок за 5 минут работы).

- Точность и частота опроса датчиков (частота не менее 1 Гц).

- Корректность управления GPIO (включение/выключение светодиода по заданному алгоритму).

- Эффективность использования памяти (статический анализ размера прошивки).

### Необходимое оборудование и программное обеспечение

- Микроконтроллерная плата ESP32-S3-DevKitC-1 или аналогичная.

- Датчик температуры и влажности (например, DHT11/DHT22) или цифровой датчик (например, BME280).

- Датчик освещенности (фоторезистор в составе делителя напряжения или цифровой датчик, например, BH1750).

- Светодиод с токоограничивающим резистором (220-330 Ом) и макетная плата.

- Персональный компьютер с установленной ОС (Windows/Linux/MacOS).

- Среда разработки: PlatformIO (расширение для VSCode) или ESP-IDF (Command Line + VSCode плагин).

- Утилита для мониторинга последовательного порта (встроенный монитор PlatformIO или PuTTY/сериал-терминал).

- Осциллограф или логический анализатор (опционально, для проверки таймингов GPIO).

### Задания для выполнения

#### 1. Настройка среды разработки и создание проекта.

- Установить расширение PlatformIO для VSCode или настроить ESP-IDF согласно официальной документации.

- Создать новый проект, выбрав в качестве платформы Espressif ESP32-S3 (например, esp32-s3-devkitc-1).

#### Настроить конфигурацию проекта (файл platformio.ini или sdkconfig)

- Установить частоту CPU (240 МГц).

- Настроить скорость обмена по UART (115200 бод).

- Выбрать схему менеджмента памяти (Flash 2MB/4MB, PSRAM опционально).

- Убедиться в успешной компиляции шаблонного проекта (src/main.cpp), прошивке на устройство и получении тестового вывода в монитор порта.

#### 2. Разработка драйвера чтения данных с сенсоров.

- Подключить датчик температуры/влажности к интерфейсу I2C или GPIO (в зависимости от типа датчика) согласно схеме.

#### Для датчиков на шине I2C (BME280/BH1750)

- Инициализировать шину I2C с корректной тактовой частотой (100 кГц или 400 кГц).

- Реализовать функцию чтения регистров датчика и преобразования сырых значений в физические величины (°C, %, Люксы).

#### Для датчика с аналоговым выходом (фоторезистор)

- Настроить АЦП (ADC) для считывания напряжения с делителя.

- Реализовать масштабирование полученного кода (0-4095) в значение освещенности в Люксах (эмпирическая формула или LUT).

- Реализовать обработку ошибок (таймауты ожидания готовности датчика, проверка контрольной суммы для DHT).

#### 3. Реализация алгоритма управления GPIO.

- Настроить порт GPIO, к которому подключен светодиод, в режим OUTPUT.

#### Разработать логику управления

- Если значение освещенности (с фоторезистора) падает ниже заданного порога (например, 50 Люкс) – включить светодиод.

- Если значение освещенности выше порога – выключить светодиод.

- Реализовать "защиту от дребезга" для порогового переключения (введение гистерезиса), чтобы светодиод не мерцал при колебаниях освещенности около порога.

- Добавить возможность управления светодиодом через ШИМ (PWM) для плавного изменения яркости на основе значения с датчика.

#### 4. Встраивание логирования и отладки.

- Организовать вывод в последовательный порт всех измеряемых величин с временными метками (псевдо-таймер, счетчик миллисекунд).

- Реализовать обработку команд с терминала (например, команды help, state для вывода статуса датчиков, set_threshold 70 для изменения порога срабатывания).

- Проверить работу системы в различных условиях (изменение освещенности рукой, нагрев/охлаждение датчика температуры).

#### 5. Измерение и оценка показателей.

- Замерить тактовую частоту выполнения основного цикла (loop).

- Оценить загрузку CPU при помощи счетчика времени простоя.

- Проанализировать использование Flash и RAM (с помощью вывода компилятора).

- Проверить стабильность чтения датчиков в течение длительного периода (стресс-тест 30 минут).

## КИМ Р1.2. Организация питания, настройка DMA и прерываний для эффективного сбора мультимодальных данных. Обзор STM32Cube.AI для генерации C-кода

**Форма проведения:** очная.

**Продолжительность по РПД:** 3 академических часа.

### Цель работы

Освоение методов проектирования энергоэффективных систем сбора мультимодальных данных (аналоговые сигналы, цифровые интерфейсы) на базе микроконтроллеров STM32 с использованием прямого доступа к памяти (DMA) и системы прерываний. Дополнительной задачей является ознакомление с инструментом STM32Cube.AI для автоматической генерации оптимизированного C-кода нейросетей для последующего развертывания на целевом устройстве.

### Задача работы

Реализовать систему сбора данных с двух типов сенсоров (аналоговый акселерометр и цифровой микрофон на шине I2S) с использованием DMA для разгрузки CPU и прерываний для синхронизации. Настроить тактирование и режимы энергопотребления для минимизации расхода тока. Произвести обзорное ознакомление с STM32Cube.AI, выполнив конвертацию предварительно обученной модели в C-код и оценку ее размера. Метриками оптимизации будут являться:

- Эффективность работы DMA (частота прерываний заполнения буфера).

- Ток потребления в активном режиме сбора данных (mA).

- Объем тактируемых периферийных блоков, влияющий на энергопотребление.

- Размер и скорость выполнения сгенерированного STM32Cube.AI C-кода (опционально).

### Необходимое оборудование и программное обеспечение

- Отладочная плата на базе STM32 с поддержкой FPU и DSP-инструкций (например, STM32L4R9I-DISCO или STM32F407G-DISC1, либо Nucleo-64 с STM32L476RG).

- Датчик, подключаемый по аналоговому входу (например, потенциометр или тензорезистор, имитирующий датчик давления/акселерометр через АЦП).

- Цифровой микрофон с интерфейсом I2S (например, MP34DT01) для захвата аудио-данных.

- USB-амперметр (или мультиметр в режиме измерения тока) для оценки энергопотребления.

#### Программное обеспечение

- STM32CubeMX — для инициализации периферии, тактирования и генерации кода.

- IDE: Keil MDK, IAR EWARM или STM32CubeIDE (на выбор).

- STM32Cube.AI — пакет для интеграции в STM32CubeMX или отдельный инструмент.

- TensorFlow/Keras — для подготовки и обучения эталонной модели (для демонстрации конвертации).

- Логический анализатор или осциллограф — для верификации таймингов DMA и прерываний.

### Задания для выполнения

#### 1. Настройка тактирования и питания в STM32CubeMX.

- Создать проект в STM32CubeMX для выбранного МК.

#### Настроить систему тактирования (Clock Tree)

- Выбрать источник тактового сигнала (HSE — внешний кварц или HSI — внутренний RC-генератор).

- Установить тактовую частоту ядра (например, 80 МГц для STM32L4) и частоты шин APB1/APB2.

#### Настроить режимы энергопотребления

- Включить режим Sleep Mode при простое CPU.

- Настроить переход ядра в Low-Power Run Mode при снижении активности.

- Сконфигурировать регулятор напряжения на низкое потребление (для STM32L4 — Voltage Scaling).

- Сгенерировать базовый код и проверить ток потребления в состоянии "холостого хода".

#### 2. Настройка АЦП и DMA для сбора аналоговых данных.

- Подключить аналоговый сенсор к одному из входов АЦП.

#### Настроить АЦП

- Разрешение: 12 бит.

- Тактирование: синхронизировать с таймером (например, TIM2) для запуска с заданной частотой дискретизации (1 кГц).

- Режим сканирования каналов (если используется более одного входа).

#### Настроить DMA для АЦП

- Режим циклического буфера (Circular Mode).

- Размер буфера: 256 отсчетов для накопления данных.

- Настроить прерывание по заполнению половины буфера (Half Transfer) и полного буфера (Complete Transfer).

- Реализовать обработчик прерывания DMA, который запускает флаг для основного цикла обработки данных.

#### 3. Настройка I2S и DMA для сбора цифровых аудиоданных.

- Подключить цифровой микрофон к интерфейсу I2S (например, I2S2).

#### Настроить I2S

- Режим Master Receive (или Slave Receive).

- Частота дискретизации: 8 кГц или 16 кГц.

- Формат данных: 16 или 24 бита, моно.

#### Настроить второй канал DMA для I2S

- Связать с запросами I2S RX.

- Размер буфера: 512 байт (или более для борьбы с переполнением).

- Настроить прерывание по окончании приема блока данных (Complete Transfer).

- Реализовать синхронизацию двух потоков данных (аналогового и цифрового) с помощью временных меток (Time Stamping) на основе системного таймера (SysTick).

#### 4. Реализация основного алгоритма обработки и управления питанием.

- В основном цикле (while(1)) реализовать ожидание флагов от DMA-прерываний.

#### При получении флага "заполнен полубуфер/полный буфер"

- Скопировать данные из DMA-буфера в рабочий массив.

- Выполнить предварительную обработку (нормализацию, фильтрацию, усреднение для контроля порога).

- Передать данные (по желанию) через UART в монитор порта.

#### Реализовать динамическое управление питанием

- Если данные с датчиков не изменяются в течение длительного времени (3 секунды), перевести МК в Stop Mode с пробуждением от внешнего прерывания.

- При пробуждении восстановить тактирование и продолжить сбор данных.

#### 5. Ознакомление с STM32Cube.AI.

- Загрузить предварительно обученную модель в формате Keras (.h5) или ONNX (например, модель распознавания жестов на основе 3-осевого акселерометра).

#### Проверка корректности (Тест целостности данных)

- Верификация данных АЦП: Сравнение полученных значений с эталонным напряжением (подключение известного опорного напряжения).

- Верификация I2S: Проверка целостности аудиопотока (отсутствие пропусков сэмплов) с помощью подачи тестового сигнала и анализа FFT на ПК.

- Верификация DMA: Контроль заполнения буфера (отсутствие переполнения Underflow/Overflow) через статусные регистры.

- Тест энергопотребления: Сравнение потребления в Active и Stop Mode с паспортными значениями из даташита.

#### Критерии успешной проверки

- Размер DMA-буфера достаточен для обработки без потерь (подтверждено логическим анализатором).

- Ток потребления в Stop Mode не превышает 50 мкА (для STM32L4).

- Прерывания от DMA и I2S не "висят" и не блокируют системный таймер.

- Сгенерированный STM32Cube.AI код успешно собирается без ошибок линковки.

#### Чеклист для production-развертывания

- Все используемые тактовые сигналы проверены на стабильность (отклонение частоты < 1%).

- Проведена оценка времени нахождения в режиме Active/Stop для расчета срока службы батареи.

- DMA-каналы приоритезированы правильно (I2S выше, чем ADC, при необходимости).

- Обработчики прерываний (ISR) оптимизированы — быстрые входы/выходы, основной код в циклах.

- Код, сгенерированный STM32Cube.AI, прошел валидацию на тестовых наборах данных непосредственно на плате.

- В случае сбоя питания, предусмотрено автоматическое перезапуска сбора данных (Watchdog).
