# Методические указания преподавателям по использованию ресурсов

## 1. Общие положения

Настоящие методические указания разработаны для преподавателей, участвующих в разработке и проведении контрольных измерительных материалов (КИМ) по дисциплине «Инженерия встроенных систем машинного обучения». Документ содержит систематизированные рекомендации по подбору и применению аппаратных платформ, программных инструментов, облачных сервисов и методических материалов при создании лабораторных работ, оценочных заданий и проектных кейсов с учётом отраслевой специфики ТЭК и Транспорт.

Цель указаний — обеспечить единый подход к ресурсному обеспечению КИМ, соответствующему заявленным компетенциям (PL-4, DL-1, LC-5, DL-3) и современным требованиям промышленной разработки Edge AI/TinyML-решений.

### 2.2 Рекомендации по выбору платформы для лабораторных работ

**ESP32-S3 рекомендуется для:**

- Лабораторных работ 1–5 (Модули 1–3) — программирование периферии, работа с сенсорами, FreeRTOS, передача данных.

- Работ, связанных с беспроводной передачей (BLE/Wi-Fi).

- Кейсов с жесткими ограничениями энергопотребления.

**STM32 рекомендуется для:**

- Лабораторных работ по организации DMA и прерываний (Модуль 1).

- Демонстрации промышленного подхода.

- Задач классического компьютерного зрения с TinyML-инференсом.

**Rockchip RK3588/RK3576 рекомендуется для:**

- Лабораторных работ по Object Detection (RKNN-Toolkit2).

- Кейсов с требованиями высокой производительности и низкой латентности (\<50 мс).

- Задач, требующих встраиваемого Linux и контейнеризации.

### 2.3 Дополнительное оборудование

Для выполнения полного цикла лабораторных работ рекомендуется иметь в лабораторном фонде:

- **USB-амперметры** — для измерения тока в различных режимах (Модуль 6).

- **Регулируемые источники питания** — для тестирования сбоев питания.

- **Осциллографы/логические анализаторы** — для верификации таймингов и DMA-транзакций.

- **Наборы сенсоров**: BME280, MPU6050, цифровой микрофон (I2S), OV2640/OV7675 камера.

- **Актуаторы**: сервоприводы (SG90), светодиоды, драйверы двигателей.

### 3.2 Специализированные платформы

**Edge Impulse** рекомендуется как сквозная платформа для быстрого прототипирования TinyML-приложений :

- Позволяет выполнить полный пайплайн от сбора данных до развертывания без написания сложного кода.

- Поддерживает INT8-квантование и генерацию оптимизированных библиотек для ESP32-S3.

- Особенно эффективен для задач Wake Word и классификации данных с сенсоров (Модуль 5).

STM32Cube.AI — специализированный инструмент для генерации оптимизированного C-кода под STM32 :

- Интегрируется в STM32CubeMX, предоставляет анализ ресурсов (Flash/RAM) до развертывания.

- Рекомендуется для демонстрации промышленного подхода к развертыванию.

## 5. Методические материалы для разработки КИМ

### 5.1 Структура лабораторной работы

При разработке лабораторной работы рекомендуется придерживаться следующей структуры, соответствующей примеру из предоставленных материалов:

1.  **Тема и номер работы**.

2.  **Формат проведения** (продолжительность, форма).

3.  **Цель работы** — формулировка осваиваемых навыков и компетенций.

4.  **Задача работы** — конкретные практические задания с указанием метрик успешности.

5.  **Необходимое оборудование и ПО** — детальный перечень.

6.  **Задания для выполнения** — пошаговые инструкции с указанием контрольных точек.

7.  **Ожидаемые результаты** — структура отчета, перечень графиков и таблиц.

8.  **Проверка корректности** — тесты и чек-листы для самопроверки.

9.  **Контрольные вопросы** — 10 вопросов для проверки понимания.

### 5.2 Принципы разработки оценочных материалов

При разработке КИМ следует учитывать:

1.  **Соответствие компетенциям** — каждое задание должно проверять конкретный индикатор компетенции (PL-4.2, DL-1.11, LC-5.1, DL-3.3).

2.  **Отраслевая релевантность** — кейсы должны быть адаптированы под специфику ТЭК или Транспорта:

    - *ТЭК:* вибродиагностика насосов, мониторинг трубопроводов, контроль параметров скважин.

    - *Транспорт:* детекция препятствий, мониторинг состояния груза, системы безопасности.

3.  **Практическая применимость** — задания должны моделировать реальные производственные задачи с использованием промышленных инструментов.

4.  **Сквозная интеграция** — лабораторные работы должны быть взаимосвязаны, формируя полный пайплайн от сбора данных до промышленного развертывания.

5.  **Масштабируемость сложности** — задания должны иметь базовый, продвинутый и экспертный уровни для дифференциации студентов.

### 6.2 Рубежный контроль (защита пайплайна)

Для рубежного контроля рекомендуется использовать:

- **Python-скрипт конвертации** как артефакт (применение RKNN-Toolkit2/TFLite Converter).

- **Демонстрационный набор данных** (калибровочная выборка) с фиксированным составом.

- **Целевое устройство** с предустановленной прошивкой для приёма модели.

### 6.3 Промежуточная аттестация (защита проекта)

При оценке итогового проекта комиссия должна иметь доступ к:

- Работающему прототипу на целевом оборудовании.

- Исходному коду и отчёту в соответствии с чек-листом.

- Логам инференса и профилирования (включая энергопотребление).

## 7. Рекомендации по отраслевой адаптации КИМ

### 7.1 ТЭК-специфика

Для кейсов топливно-энергетического комплекса рекомендуется:

- Использовать датчики вибрации и температуры для предиктивной диагностики.

- Акцентировать внимание на энергосбережении (расчёт времени работы от батареи \>2 лет).

- Включать LoRa как альтернативу BLE/Wi-Fi для передачи на дальние расстояния.

### 7.2 Транспортная специфика

Для кейсов транспортной отрасли рекомендуется:

- Использовать камеры и ультразвуковые датчики для детекции препятствий.

- Акцентировать внимание на низкой латентности (\<50 мс).

- Использовать Rockchip NPU для Object Detection в реальном времени.

## 8. Чек-лист готовности ресурсов для проведения КИМ

### 8.1 Перед началом семестра

- Проверено наличие необходимых аппаратных платформ (ESP32-S3, STM32, Rockchip).

- Установлены и протестированы инструменты разработки (PlatformIO, Arduino IDE, Python, RKNN-Toolkit2).

- Проверен доступ к облачным сервисам (Edge Impulse, MQTT-брокеры).

- Подготовлены шаблоны проектов для лабораторных работ.

- Проверена работоспособность измерительного оборудования (амперметры, осциллографы).

### 8.2 Перед каждой лабораторной работой

- Проверена актуальность версий библиотек и инструментов.

- Протестированы примеры кода на целевых платформах.

- Подготовлены калибровочные и тестовые наборы данных.

- Проверено наличие расходных материалов (провода, резисторы, макетные платы).

### 8.3 Перед защитой проектов

- Определён состав апелляционной комиссии.

- Подготовлены оценочные листы (рубрики).

- Проверена работоспособность демонстрационного оборудования.
