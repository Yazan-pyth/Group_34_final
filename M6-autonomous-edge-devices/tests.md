## Л6.1. Энергетический бюджет, Deep Sleep, watchdog, измерение мощности

### 1. Устройство 1 с в active (100 мА) и 59 с в sleep (0,1 мА) каждую минуту. Какой режим доминирует в среднем потреблении?

- **А)** Определить невозможно без напряжения
- **Б)** Sleep — потому что длится дольше по времени
- **В)** Оба вносят поровну
- **Г)** Active — короткий, но с большим током, вносит основной вклад

### 2. Как корректно оценить среднее потребление устройства с несколькими режимами?

- **А)** Взять максимальный ток по всем режимам
- **Б)** Взять только ток инференса
- **В)** Взять минимальный ток
- **Г)** Взвесить токи режимов по их доле времени (duty cycle)

### 3. Wi-Fi-передача заметно бьёт по автономности батарейного датчика, потому что:

- **А)** Wi-Fi уменьшает Flash
- **Б)** Wi-Fi запрещает sleep аппаратно
- **В)** Установление соединения и передача дают большой пиковый ток
- **Г)** Wi-Fi меняет число классов модели

### 4. Почему нельзя доверять только паспортному энергопотреблению при оценке автономности?

- **А)** Паспорт всегда занижает значение вдвое
- **Б)** Реальная нагрузка, конфигурация и duty cycle отличаются от паспортных условий
- **В)** Паспорт не содержит напряжения
- **Г)** Паспорт применим только к Rockchip

### 5. Что такое brownout?

- **А)** Сбой из-за просадки напряжения питания ниже порога
- **Б)** Перегрев NPU
- **В)** Падение точности после квантования
- **Г)** Переполнение DMA-буфера

### 6. Почему важен именно пиковый (а не только средний) ток?

- **А)** Он влияет только на логирование
- **Б)** Он не связан со стабильностью работы
- **В)** Источник питания должен выдерживать кратковременные пики без просадки
- **Г)** Он определяет число классов

### 7. Что такое throttling у SoC вроде RK3588?

- **А)** Ускорение NPU под нагрузкой
- **Б)** Снижение частоты из-за перегрева или просадки питания
- **В)** Метод сжатия модели
- **Г)** Режим OTA-обновления

### 8. Слишком частое пробуждение из sleep невыгодно по энергии, потому что:

- **А)** Растут накладные расходы на запуск, инициализацию и связь
- **Б)** Уменьшается размер прошивки
- **В)** Повышается точность модели
- **Г)** Отключается watchdog

### 9. Какие программные меры реально снижают энергопотребление на MCU?

- **А)** Увеличение размера модели
- **Б)** Повышение частоты опроса сенсоров
- **В)** Снижение тактовой частоты, отключение неиспользуемой периферии, использование sleep
- **Г)** Постоянно активный Wi-Fi

### 10. Как достоверно подтвердить заявленную автономность устройства?

- **А)** Взять цифру из рекламного описания платы
- **Б)** Рассчитать по бюджету и подтвердить измерениями в реальном duty cycle
- **В)** Измерить только ток в sleep-режиме
- **Г)** Оценить по числу параметров модели
