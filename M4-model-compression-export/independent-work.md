## 2. Теоретические основы

### 2.1 Квантование нейронных сетей

Квантование — метод сжатия моделей, заключающийся в уменьшении разрядности весов и активаций (например, переход от 32-битных чисел с плавающей запятой к 8-битным целым). Это позволяет:

- **Сократить размер модели** — использование INT8 вместо FP32 уменьшает объём памяти в 4 раза.

- **Ускорить инференс** — целочисленные операции выполняются быстрее, особенно на устройствах с аппаратной поддержкой INT8 (NPU Rockchip, DSP ESP32-S3).

- **Снизить энергопотребление** — меньше обращений к памяти и более эффективные вычисления.

**Пост-тренировочное квантование (PTQ)** выполняется после обучения модели и не требует дообучения. Для INT8-квантования необходим **калибровочный набор данных** — репрезентативная выборка, на основе которой определяются масштабирующие коэффициенты (scale) и нулевые точки (zero_point) для каждого тензора.

### 2.2 Прунинг нейронных сетей

Прунинг — метод удаления «избыточных» весов или нейронов, которые вносят наименьший вклад в выход модели.

| Тип прунинга      | Описание                                             | Преимущества                                        | Недостатки                                                  |
|-------------------|------------------------------------------------------|-----------------------------------------------------|-------------------------------------------------------------|
| **Неструктурный** | Удаление отдельных весов (обнуление малых по модулю) | Высокая степень сжатия, минимальная потеря точности | Создаёт разрежённые матрицы, плохо поддерживается аппаратно |
| **Структурный**   | Удаление целых каналов, фильтров или слоёв           | Аппаратно-эффективен, ускоряет инференс             | Более грубое сжатие, большее падение точности               |

Для Edge AI-устройств предпочтительнее **структурный прунинг**, так как он создаёт регулярную разрежённость, хорошо поддерживаемую аппаратными ускорителями.

### 2.3 Комбинированное сжатие

Наибольший эффект достигается при комбинировании методов: например, структурный прунинг + INT8-квантование. Такой подход позволяет одновременно уменьшить размер модели, ускорить инференс и сохранить приемлемую точность.

### Этап 2. Применение методов сжатия (4 ч)

**Задание:**  
Реализовать пайплайн сжатия модели с использованием INT8-квантования и/или прунинга.

**2.1. INT8-квантование через TFLite Converter (для MCU):**

python

import tensorflow as tf

*\# Загрузка модели*

converter = tf.lite.TFLiteConverter.from_saved_model('model_saved')

*\# Настройка INT8-квантования*

def representative_dataset():

for \_ in range(100):

data = np.random.rand(1, 128, 128, 3).astype(np.float32)

yield \[data\]

converter.representative_dataset = representative_dataset

converter.target_spec.supported_ops = \[tf.lite.OpsSet.TFLITE_BUILTINS_INT8\]

converter.inference_input_type = tf.int8

converter.inference_output_type = tf.int8

*\# Конвертация*

tflite_model = converter.convert()

**2.2. INT8-квантование через RKNN-Toolkit2 (для NPU Rockchip):**

python

from rknn.api import RKNN

rknn = RKNN()

*\# Загрузка ONNX-модели*

rknn.load_onnx(model='model.onnx')

*\# Конфигурация квантования INT8 с калибровкой*

rknn.config(target_platform='rk3588', batch_size=1)

rknn.build(do_quantization=True, dataset='calibration_dataset.txt')

*\# Экспорт RKNN-модели*

rknn.export_rknn('model_int8.rknn')

**Важно:** для INT8-квантования критически важен **калибровочный набор данных** — он должен быть репрезентативным для целевого приложения. Рекомендуемый размер — 100–500 примеров.

**2.3. Структурный прунинг (опционально, для MCU):**

Использовать TensorFlow Model Optimization Toolkit:

python

import tensorflow_model_optimization as tfmot

*\# Применение структурного прунинга*

pruning_schedule = tfmot.sparsity.keras.PolynomialDecay(

initial_sparsity=0.0, final_sparsity=0.5, begin_step=0, end_step=1000

)

model_pruned = tfmot.sparsity.keras.prune_low_magnitude(

model, pruning_schedule=pruning_schedule

)

**Результат этапа:**

- Сжатая модель в формате .tflite (для MCU) или .rknn (для NPU).

- Логи процесса конвертации с указанием размера модели до и после сжатия.
