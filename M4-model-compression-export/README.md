## КИМ Р4.1. Квантование модели до INT8, анализ потери качества, подбор калибровочной выборки, замеры на целевом железе

**Форма проведения:** очная.

**Продолжительность по РПД:** 3 академических часа.

### Цель работы

Практическое освоение методов пост-тренировочного квантования моделей глубокого обучения с плавающей запятой (float32) до целочисленного формата INT8, изучение влияния квантования на точность модели, освоение методик подбора репрезентативной калибровочной выборки и оценка реальной производительности квантованной модели на целевой аппаратной платформе (ESP32-S3 или STM32).

### Задача работы

Выполнить полный цикл оптимизации ML-модели для развертывания на ресурсоограниченном устройстве: выбрать модель (например, для распознавания жестов на основе данных акселерометра или классификации изображений MNIST/Fashion-MNIST), обучить эталонную float32-модель, провести пост-тренировочное квантование в INT8 с использованием различных калибровочных выборок, проанализировать потерю точности, выбрать оптимальную стратегию квантования и развернуть модель на целевом устройстве с замерами реальной производительности (время инференса, потребление памяти, энергопотребление). Метриками оптимизации будут являться:

- Падение точности (Accuracy Drop) после квантования относительно float32-модели (абсолютное и относительное).

- Размер модели в формате float32 vs INT8 (коэффициент сжатия).

- Время выполнения инференса (inference latency) на целевом устройстве.

- Использование памяти (Flash/RAM) при развертывании.

- Стабильность квантованных весов и активаций (отсутствие "выбросов" quant-параметров).

### Необходимое оборудование и программное обеспечение

- Микроконтроллерная плата: ESP32-S3-DevKitC-1 (с поддержкой векторных инструкций) или STM32F746G-DISCO/NUCLEO-L4R5ZI (с FPU).

- Датчики (акселерометр/гироскоп) для сбора данных в реальном времени для финального тестирования.

- Персональный компьютер с установленной ОС и достаточным объемом ОЗУ (не менее 8 ГБ для работы с датасетами).

- Программное обеспечение для обучения исходной модели: Python 3.8+, TensorFlow 2.x/Keras, NumPy, Pandas, Matplotlib.

#### Программное обеспечение для квантования и конвертации

- TensorFlow Lite Converter (TFLite) — для пост-тренировочного квантования (PTQ) в INT8.

- TensorFlow Model Optimization Toolkit (для QAT — опционально).

- xxd утилита или Python-скрипт для конвертации модели в C-массив.

- Среда разработки для целевого железа: PlatformIO + Arduino для ESP32-S3 (с библиотекой TensorFlowLite_ESP32 или TensorFlow Lite Micro).

- Калибровочный датасет: подмножество данных, репрезентативное для целевого приложения (например, 100-200 случайных примеров из тренировочной выборки).

- Утилиты для мониторинга последовательного порта и вывода результатов.

- USB-амперметр (опционально) для замера энергопотребления во время инференса.

### Задания для выполнения

#### 1. Подготовка эталонной float32-модели.

#### Выбрать задачу классификации, подходящую для TinyML

- Распознавание жестов на основе 3-осевого акселерометра (датчик MPU6050) — набор данных UCI Gesture Recognition или собственный.

- Классификация рукописных цифр (MNIST) или предметов одежды (Fashion-MNIST) с разрешением 28x28.

- Распознавание голосовых команд (Keyword Spotting) — например, набор данных Google Speech Commands (подмножество).

- Загрузить и предобработать данные: нормализация, разделение на train/val/test.

#### Построить и обучить модель в Keras/TensorFlow

- Архитектура: сверточная сеть (CNN) или полносвязная сеть с ограниченным числом параметров (< 100K).

- Использовать активации ReLU или ReLU6 (предпочтительнее для квантования).

- Достичь точности на тестовой выборке не менее 90% (для классификации).

- Сохранить модель в формате .h5 и сконвертировать в TensorFlow SavedModel или .tflite (float32).

#### 2. Подбор и подготовка калибровочной выборки.

#### Сформировать калибровочный датасет из тренировочной выборки

- Объем: 100, 200, 500 или 1000 репрезентативных примеров.

- Критерии репрезентативности: разнообразие классов, статистическое распределение, близкое к тестовой выборке (проверить через визуализацию t-SNE или распределение активаций).

#### Сравнить влияние размера калибровочной выборки на потерю точности

- Провести квантование с калибровкой на 100, 200, 500, 1000 выборках.

- Зафиксировать полученную точность INT8-модели после квантования.

- Визуализировать зависимость Accuracy от размера калибровочного набора.

- Реализовать функцию-генератор калибровочных данных для подачи в TFLite Converter.

#### 3. Пост-тренировочное квантование (PTQ) в INT8.

#### Выполнить квантование float32-модели в INT8 с использованием TFLiteConverter

- Вариант 1: tf.lite.Optimize.DEFAULT (веса и активации в INT8) с калибровкой.

- Вариант 2: tf.lite.Optimize.DEFAULT для весов + активации float16 (для сравнения).

- Вариант 3: квантование только весов (weight-only) — активации в float32 (для сравнения).

- Сохранить квантованные модели в формате .tflite и оценить размер файлов.

- Извлечь квант-параметры (scale и zero_point) для входных и выходных тензоров.

#### 4. Оценка падения качества (Accuracy Drop).

- Загрузить эталонную float32-модель и все квантованные варианты.

#### Выполнить оценку точности на тестовой выборке

- Для float32-модели — эталонная точность.

- Для INT8-модели — точность с учетом квантования (использовать interpreter для инференса).

- Вычислить абсолютное и относительное падение точности для каждого варианта.

- Построить график сравнения точности float32 vs INT8.

#### Для моделей с наибольшей потерей точности выполнить анализ

- Послойное распределение весов и активаций.

- Выявление "чувствительных" слоев (с большим разбросом квант-параметров).

- Предложение методов для снижения потерь (QAT — квантование с обучением, изменение архитектуры, использование симметричного vs асимметричного квантования).

#### 5. Развертывание INT8-модели на целевом железе.

- Конвертировать квантованную .tflite-модель в C-массив с помощью xxd или Python-скрипта.

#### Интегрировать массив в проект для ESP32-S3 (или STM32)

- Подключить библиотеку TensorFlow Lite Micro (или EloquentTinyML).

- Инициализировать интерпретатор с моделью, выделить память для тензоров.

- Загрузить тестовые данные из массива или с реального датчика во входной тензор.

- Выполнить инференс, считать результаты выходного тензора.

- Организовать вывод в UART результатов (класс, вероятность) вместе с временем выполнения.

#### 6. Замеры производительности на целевом железе.

#### Замерить ключевые метрики на целевом устройстве

- Время инференса (Latency): Использовать esp_timer_get_time() для ESP32-S3 или HAL_GetTick() для STM32 — замерить для 100 последовательных запусков, вычислить среднее, минимальное и максимальное.

- Использование памяти (Flash/RAM): Сравнить с эталонной float32-моделью (если она еще помещается).

- Потребление тока: Использовать USB-амперметр, измерить ток в режиме инференса и в режиме ожидания, вычислить энергию на один инференс.

#### Построить сравнительные таблицы

- Размер модели: float32 (без квантования) vs INT8 (с квантованием) в КБ и коэффициент сжатия.

- Точность: float32 vs INT8 (падепние точности в процентах).

- Время инференса: float32 (если запускается) vs INT8 в мс.

- Сравнить с теоретическими ожиданиями (ускорение в 2-4 раза, сжатие в ~4 раза).

#### Проверка корректности (Тест целостности данных)

- Верификация калибровочной выборки: Проверка, что она не содержит примеров из тестовой выборки (Data Leakage) — использовать пересечение индексов.

- Верификация квантованной модели: Сравнение инференса на 10 случайных примерах между float32 и INT8 в Python (передача через интерпретатор) и на целевом устройстве.

- Тест на репрезентативность: Сравнение распределения калибровочной выборки с тестовой (использовать Kullback–Leibler Divergence).

- Тест на переполнение (Overflow): Проверка, что при инференсе на экстремальных значениях (выходящих за диапазон калибровки) не возникает численных артефактов (NaN/Inf).

- Тест стабильности на железе: Выполнить 500 последовательных инференсов на реальном устройстве, убедиться в отсутствии изменений результатов со временем.

#### Критерии успешной проверки

- Падение точности INT8-модели относительно float32 не превышает 2% для задач классификации (при калибровке на 500+ примерах).

- Размер модели уменьшен не менее чем в 3.5 раза (с учетом заголовков TFLite).

- Время инференса на ESP32-S3 не превышает 200 мс для модели с 5-10 слоями.

- Отсутствие переполнения стека и памяти на устройстве при 100+ запусках.

#### Чеклист для production-развертывания

- Модель прошла валидацию на реальных данных с датчика, а не только на синтетическом тестовом наборе.

- Калибровочная выборка собрана в условиях, максимально приближенных к эксплуатационным (различные освещения, температура, наклон датчика).

- Проведен анализ на аномалии: проверка влияния выбросов в калибровочной выборке на точность модели (Robustness Test).

- Используется квантование по каналам (per-channel quantization) для улучшения точности (если поддерживается TFLite).

- На целевой платформе включены аппаратные ускорители (например, DSP-инструкции ESP32-S3 или CMSIS-NN для STM32) для ускорения инференса.

- Размер модели не превышает выделенной области Flash с запасом 20% на будущие обновления.

- Проведен стресс-тест при пониженном напряжении питания (проверка стабильности квантованных вычислений).

## КИМ Р4.2. Автоматизация пайплайна сжатия и профилирование инференса на целевом железе ESP32-S3

**Форма проведения:** очная.

**Продолжительность по РПД:** 3 академических часа.

### Цель работы

Освоение методов автоматизации полного пайплайна сжатия нейросетевых моделей (квантование, оптимизация архитектуры, удаление "мертвых" нейронов) с использованием инструментов командной строки и скриптов, а также получение навыков профилирования производительности инференса непосредственно на целевом микроконтроллере ESP32-S3 с использованием встроенных аппаратных счетчиков производительности (Performance Counters) и системных таймеров.

### Задача работы

Разработать автоматизированный скриптовый пайплайн (на Python или Bash), который принимает на вход обученную модель в формате Keras/TFLite, выполняет её многовариантное сжатие с различными стратегиями (квантование INT8, обрезка весов/Pruning, кластеризация весов), оценивает метрики качества и размера, а затем развертывает сжатые варианты на ESP32-S3 и выполняет детальное профилирование инференса с фиксацией времени выполнения по слоям, использования памяти и энергопотребления. Метриками оптимизации будут являться:

- Время выполнения полного пайплайна сжатия (сек).

- Падение точности (Accuracy Drop) для каждого метода сжатия.

- Коэффициент сжатия модели (размер float32 / размер сжатой модели).

- Время инференса на ESP32-S3 для каждого варианта сжатия.

- Профиль выполнения по слоям (время на каждый слой/операцию) — для выявления "узких мест".

- Энергопотребление на один инференс (мДж).
