# Методические указания для самостоятельной работы студента по теме «Изучение фреймворков OpenCV, MediaPipe, TFLite, RKNN, подготовка отчетов по лабораторным работам. Решение кейса от индустриального партнера: разработка модуля детекции препятствий или аномалий звука для целевой платформы»

**Дисциплина:** Инженерия встроенных систем машинного обучения  
**Модуль:** 5 (Алгоритмы компьютерного зрения и обработки сигналов на периферии)  
**Компетенция:** DL-3 (средний уровень)  
**Объём СРС:** 10 академических часов

## 1. Цель и задачи

**Цель:** Сформировать у обучающихся практические навыки применения фреймворков компьютерного зрения и машинного обучения (OpenCV, MediaPipe, TensorFlow Lite, RKNN) для разработки модулей детекции препятствий или аномалий звука на встроенных устройствах с учётом отраслевой специфики ТЭК и Транспорта.

**Задачи:**

- Изучить архитектуру и возможности фреймворков OpenCV, TFLite, RKNN для задач компьютерного зрения и обработки сигналов.

- Освоить методы интеграции CV- и audio-моделей на целевых платформах (ESP32-S3, Rockchip NPU).

- Разработать модуль детекции препятствий (компьютерное зрение) или детекции аномалий звука для выбранного отраслевого кейса.

- Провести профилирование производительности разработанного модуля на целевом устройстве.

- Подготовить отчёты по лабораторным работам Модуля 5.

## 2. Теоретические основы

### 2.1 Обзор фреймворков для Edge AI

| Фреймворк                 | Назначение                                                     | Целевые платформы                  | Ключевые особенности                         |
|---------------------------|----------------------------------------------------------------|------------------------------------|----------------------------------------------|
| **OpenCV**                | Компьютерное зрение (обработка изображений, детекция, трекинг) | ESP32-S3, Linux, Android           | Лёгкая версия доступна как компонент ESP-IDF |
| **MediaPipe**             | ML-пайплайны для видео/аудио в реальном времени                | Android, iOS, Python, Raspberry Pi | Поддержка GPU/TPU ускорения                  |
| **TensorFlow Lite Micro** | Выполнение ML-моделей на MCU                                   | ESP32-S3, STM32, Arduino           | Размер \< 250 КБ для типовых моделей         |
| **RKNN-Toolkit2**         | Конвертация и выполнение моделей на NPU Rockchip               | RK3588, RK3566, RK3568             | INT8-квантование, до 6 TOPS                  |

### 2.2 Архитектуры моделей для Edge AI

**Для детекции препятствий (CV):**

- **FOMO (Faster Objects, More Objects)** — лёгкая архитектура для детекции объектов на MCU, разработанная Edge Impulse. Позволяет выполнять детекцию на ESP32-CAM и ESP32-S3 с использованием bounding boxes.

- **YOLOv5/v8-nano** — оптимизированные версии для NPU Rockchip.

- **MobileNetV3 + SSD** — классическая связка для мобильных и встраиваемых устройств.

**Для детекции аномалий звука:**

- **Wake Word Detection** — распознавание ключевых слов на ESP32-S3 с TFLite Micro.

- **Audio Classification** — классификация звуковых событий (стук, скрежет, шум) с использованием MFCC-признаков.

## 3. Этапы выполнения

### Этап 1. Изучение фреймворков (2 ч)

**Задание:**  
Самостоятельно изучить архитектуру и возможности каждого фреймворка.

**1.1. OpenCV на ESP32-S3:**

- OpenCV доступен как компонент для ESP-IDF.

- Поддерживает извлечение 2D-признаков, обработку изображений.

- ESP32-S3 имеет выделенный интерфейс CAM DVP со скоростью в 2-3 раза выше, чем у ESP32.

- Рекомендуется использовать DVP-камеры для ESP32-S3.

**1.2. MediaPipe:**

- Google AI Edge фреймворк для низколатентных ML-пайплайнов.

- Поддерживает задачи: генеративный AI, компьютерное зрение, текст, аудио.

- Работает на CPU, GPU и TPU.

**1.3. TensorFlow Lite Micro на ESP32-S3:**

- Официальный репозиторий Espressif с примерами.

- Пример person_detection — модель 250 КБ для распознавания людей на изображениях с камеры.

- Поддерживает распознавание ключевых слов (Wake Word).

**1.4. RKNN-Toolkit2:**

- Конвертация моделей из PyTorch/TensorFlow/ONNX в RKNN.

- Процесс: на ПК конвертация → на устройстве выполнение через Python API.

- Рекомендуется opset 11-13 для стабильности.

**Результат этапа:** Конспект (3–5 стр.) с описанием:

- Архитектуры и назначения каждого фреймворка.

- Поддерживаемых платформ и моделей.

- Примеров использования.

### Этап 2. Выбор кейса и анализ требований (1.5 ч)

**Задание:**  
Выбрать один из двух отраслевых кейсов и определить требования к модулю.

**Вариант А. Транспорт — модуль детекции препятствий:**

- **Целевая платформа:** ESP32-S3 с камерой (DVP) или Rockchip RK3588 с NPU.

- **Задача:** Обнаружение препятствий (люди, объекты) в реальном времени.

- **Требования:** латентность \< 50 мс (для RK3588) или \< 200 мс (для ESP32-S3), точность \> 85%.

- **Инструменты:** OpenCV (предобработка) + TFLite Micro / RKNN (инференс).

**Вариант Б. ТЭК — модуль детекции аномалий звука:**

- **Целевая платформа:** ESP32-S3 с I2S-микрофоном.

- **Задача:** Обнаружение аномальных звуков (стуки, скрежет, утечки) на насосном оборудовании.

- **Требования:** частота дискретизации 16 кГц, латентность \< 100 мс, точность \> 90%.

- **Инструменты:** TFLite Micro для аудиоклассификации.

**Результат этапа:** Техническое задание (1–2 стр.) с:

- Описанием решаемой задачи.

- Выбором целевой платформы и обоснованием.

- Требованиями к производительности и точности.

### Этап 3. Разработка пайплайна (3 ч)

**Задание:**  
Разработать пайплайн обработки данных и выполнения инференса для выбранного кейса.

**3.1. Для детекции препятствий (CV):**

**Архитектура пайплайна:**

text

Камера (DVP/MIPI) → Захват кадра → Предобработка (OpenCV) →

ML-инференс (TFLite Micro / RKNN) → Постобработка → Вывод (UART/GPIO)

**Реализация на ESP32-S3 (TFLite Micro):**

- Использовать компонент esp-tflite-micro.

- Пример person_detection — основа для детекции.

- Для детекции нескольких классов использовать FOMO-архитектуру.

- ESP32-S3 поддерживает аппаратное AI-ускорение.

**Реализация на Rockchip NPU (RKNN):**

- Конвертация YOLOv5/v8 в RKNN.

- Использование rknn-toolkit2 для конвертации.

- Выполнение инференса через Python API на устройстве.

**3.2. Для детекции аномалий звука:**

**Архитектура пайплайна:**

text

Микрофон (I2S) → Захват аудио → Извлечение MFCC →

ML-инференс (TFLite Micro) → Классификация → Вывод

**Реализация:**

- Использовать I2S-микрофон на ESP32-S3.

- Предобработка: извлечение MFCC-признаков (13-20 коэффициентов).

- Модель: свёрточная или полносвязная сеть для классификации звуков.

- TFLite Micro для выполнения инференса.

**Результат этапа:**

- Код пайплайна (Python/C++).

- Схема обработки данных.

- Описание архитектуры модели.

### Этап 4. Профилирование и оптимизация (1.5 ч)

**Задание:**  
Провести замеры производительности разработанного модуля.

**Метрики для замера:**

| Метрика           | Способ замера                            | Целевое значение                     |
|-------------------|------------------------------------------|--------------------------------------|
| Время инференса   | esp_timer_get_time() / RKNN Runtime logs | \< 200 мс (ESP32), \< 50 мс (RK3588) |
| Использование RAM | esp_get_free_heap_size()                 | \< 80% доступной                     |
| Размер модели     | Размер .tflite / .rknn файла             | \< 1 МБ (ESP32), \< 10 МБ (RK3588)   |
| FPS (для CV)      | Количество кадров в секунду              | \> 5 FPS (ESP32), \> 15 FPS (RK3588) |
| Точность          | Тестовая выборка                         | \> 85%                               |

**Результат этапа:** Таблица с результатами профилирования.

    - Графики (FPS, latency, accuracy).

    - Достигнутые показатели.
