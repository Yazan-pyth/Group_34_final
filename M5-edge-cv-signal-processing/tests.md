## Л5.1. Компактные архитектуры: MobileNetV3, EfficientNet-Lite, YOLOv8-nano

### 1. За счёт чего MobileNet экономичнее классических CNN по вычислениям?

- **А)** За счёт увеличенного разрешения входа
- **Б)** За счёт полного отказа от свёрток
- **В)** За счёт работы без квантования
- **Г)** За счёт depthwise separable convolutions

### 2. Почему одних лишь FLOPs недостаточно для оценки edge-модели?

- **А)** FLOPs измеряют только точность
- **Б)** FLOPs применимы только к CPU
- **В)** FLOPs не учитывают память активаций, реальный runtime и поддержку операторов
- **Г)** FLOPs зависят от имени модели

### 3. Detection-модель показывает высокий mAP на валидации, но тормозит на NPU. Наиболее вероятная причина:

- **А)** Слишком короткое имя модели
- **Б)** mAP не может быть высоким на edge
- **В)** Модель обучена в PyTorch
- **Г)** Часть операторов уходит в CPU-fallback, снижая FPS

### 4. Что показывает метрика IoU при оценке детекции?

- **А)** Объём памяти под активации
- **Б)** Среднюю точность классификации
- **В)** Число кадров в секунду
- **Г)** Отношение площади пересечения боксов к площади их объединения

### 5. Что делает pointwise-свёртка (1×1) в связке с depthwise?

- **А)** Меняет тактовую частоту CPU
- **Б)** Смешивает информацию между каналами
- **В)** Выбирает финальные классы
- **Г)** Управляет питанием NPU

### 6. Почему большие карты активаций проблематичны для edge?

- **А)** Они требуют много RAM и пропускной способности памяти
- **Б)** Они уменьшают число классов
- **В)** Они отключают квантование
- **Г)** Они увеличивают число GPIO

### 7. Для чего в детекторе служит блок neck (напр., FPN)?

- **А)** Загрузка операционной системы
- **Б)** Буферизация кадров камеры
- **В)** Объединение признаков разных масштабов
- **Г)** Управление питанием

### 8. При выборе архитектуры под конкретный NPU решающим является совокупность:

- **А)** Accuracy, latency, память и поддерживаемость операторов на целевом железе
- **Б)** Только число слоёв
- **В)** Только год публикации
- **Г)** Только размер ONNX-файла

### 9. Как разрешение входа влияет на детектор на edge?

- **А)** Влияет только на размер файла изображения
- **Б)** Не влияет на задержку инференса
- **В)** Влияет одновременно на качество и на вычислительную стоимость
- **Г)** Влияет только на число классов

### 10. Для чего в пайплайне детекции применяется NMS (non-maximum suppression)?

- **А)** Для нормализации входа
- **Б)** Для квантования весов
- **В)** Для удаления дублирующих пересекающихся боксов одного объекта
- **Г)** Для аугментации данных

## Л5.2. Edge-задачи: OCR, Wake Word, аудиоклассификация и детекция

### 1. Wake-word система на MCU часто срабатывает без ключевой фразы. По каким метрикам её корректно оценивать?

- **А)** Только число классов
- **Б)** Только accuracy на студийном датасете
- **В)** Только размер модели
- **Г)** FAR/FRR (ложные срабатывания и пропуски), latency и энергопотребление

### 2. Почему wake-word задача хорошо ложится на MCU?

- **А)** Требует большой языковой модели
- **Б)** Работает только с изображениями
- **В)** Не использует аудиосигнал вообще
- **Г)** Модель может быть компактной и работать постоянно при малом потреблении

### 3. OCR-модель отлично работает в лаборатории, но плохо — на целевой камере устройства. Причина:

- **А)** Камера меняет архитектуру модели
- **Б)** OCR исполняется только на CPU
- **В)** Оптика, разрешение и условия съёмки отличаются от обучающих
- **Г)** OCR не поддерживает изображения

### 4. Что такое false negative для wake-word детектора?

- **А)** Система сработала без произнесённой фразы
- **Б)** Система не распознала реально произнесённую ключевую фразу
- **В)** Система выдала два класса сразу
- **Г)** Система ушла в sleep

### 5. Почему аудио обрабатывают перекрывающимися окнами фиксированной длины?

- **А)** Чтобы увеличить объём Flash
- **Б)** Чтобы уменьшить число классов
- **В)** Сигнал непрерывен, а модель принимает фрагменты фиксированного размера
- **Г)** Чтобы отключить нормализацию

### 6. Для детекции препятствий у беспилотного погрузчика приоритетны:

- **А)** Высокая полнота (recall) при ограниченной задержке
- **Б)** Только высокий precision любой ценой
- **В)** Только минимальный размер модели
- **Г)** Только число классов

### 7. Почему ограниченная (узкая) обучающая выборка опасна для edge-модели?

- **А)** Она всегда даёт высокую точность
- **Б)** Она уменьшает latency
- **В)** Она может не покрывать реальные условия эксплуатации
- **Г)** Она увеличивает footprint

### 8. Как повысить устойчивость аудиомодели к реальным условиям?

- **А)** Добавить в обучение реальные шумы и сложные сценарии
- **Б)** Убрать нормализацию входа
- **В)** Сократить длину имени класса
- **Г)** Уменьшить частоту дискретизации

### 9. Что такое edge-case в контексте валидации модели?

- **А)** Обычный типовой пример из обучающей выборки
- **Б)** Формат сериализации модели
- **В)** Редкий, но важный сценарий эксплуатации
- **Г)** Режим энергосбережения

### 10. Почему лабораторной метрики недостаточно для допуска модели в продуктив?

- **А)** Она всегда заведомо неверна
- **Б)** Она измеряет только память
- **В)** Она может не отражать реальные условия работы устройства
- **Г)** Она применима только к аудио
