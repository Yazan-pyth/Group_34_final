## КИМ Р5.1. Освоение практического пайплайна развертывания и выполнения предобученной нейросети для классификации данных с датчиков / Wake Word через Edge Impulse / TFLite Micro

**Форма проведения:** очная.

**Продолжительность по РПД:** 2 академических часа.

### Цель работы

Практическое освоение сквозного пайплайна разработки и развертывания моделей машинного обучения для ресурсоограниченных устройств с использованием платформы Edge Impulse и TensorFlow Lite Micro. В рамках работы выполняется сбор данных с датчиков (акселерометр для классификации жестов или микрофон для распознавания ключевого слова — Wake Word), обучение и оптимизация модели, ее развертывание на целевом микроконтроллере ESP32-S3 и оценка производительности в реальных условиях. Особое внимание уделяется практическим аспектам работы в экосистеме Edge Impulse, включая обработку сигналов, подбор архитектуры и квантование модели.

### Задача работы

Реализовать полный пайплайн разработки TinyML-приложения на базе ESP32-S3 с использованием Edge Impulse Studio:

- Выбрать сценарий: классификация данных с датчика (например, распознавание жестов по данным акселерометра) или распознавание ключевого слова (Keyword Spotting / Wake Word Detection) на основе аудиоданных.

- Выполнить сбор и разметку датасета через Edge Impulse Studio (с использованием встроенного датчика платы или внешнего микрофона).

- Спроектировать "импульс" (Impulse): выбрать блок обработки сигналов (MFCC для аудио, Spectral Analysis для акселерометра) и нейросетевой классификатор.

- Обучить модель, выполнить квантование в INT8, оценить качество на тестовой выборке.

- Развернуть модель на ESP32-S3 через генерацию оптимизированной C++ библиотеки (TFLite Micro или EON Compiler).

- Выполнить инференс на реальных данных и оценить задержку, точность и потребление ресурсов. Метриками оптимизации будут являться:

- Точность модели на тестовой выборке (Accuracy).

- Размер модели (Flash-память) и используемая RAM (Tensor Arena).

- Время инференса на ESP32-S3 (латентность).

- Уверенность модели в предсказаниях (Confidence Score).

- Коэффициент ложных срабатываний (False Positive Rate) — для Wake Word задач.

### Необходимое оборудование и программное обеспечение

- Микроконтроллерная плата ESP32-S3-DevKitC-1 (или аналогичная с поддержкой микрофона / акселерометра).

- Для сценария Wake Word: встроенный или внешний цифровой микрофон (подключенный по I2S).

- Для сценария классификации жестов: встроенный акселерометр (на многих платах ESP32-S3 отсутствует, может потребоваться внешний модуль MPU6050/ADXL345).

- Персональный компьютер с установленной ОС и веб-браузером (Chrome/Firefox) для работы с Edge Impulse Studio.

- Аккаунт на платформе Edge Impulse (бесплатный тариф).

- Среда разработки: PlatformIO (расширение для VSCode) с инструментарием для ESP32-S3.

- Edge Impulse CLI (инструмент командной строки) для сбора данных с устройства и развертывания.

- Утилита для мониторинга последовательного порта (Serial Monitor PlatformIO).

- Дополнительно: смартфон с микрофоном или ПК с микрофоном для записи аудиосэмплов (при работе с Wake Word).

### Задания для выполнения

#### 1. Выбор сценария и создание проекта в Edge Impulse Studio.

- Зарегистрироваться на платформе studio.edgeimpulse.com.

#### Создать новый проект

- Для Wake Word Detection: выбрать тип проекта "Audio" и указать целевое ключевое слово (например, "Hey ESP" или "ESP32").

- Для классификации жестов: выбрать тип проекта "Accelerometer data" или "Time series data".

#### Настроить параметры сбора данных

- Частота дискретизации: 16 кГц (для аудио) или 100 Гц (для акселерометра).

- Длина окна (Window Length): 1-2 секунды (для аудио — 1000-2000 мс).

- При необходимости настроить параметры DSP-обработки.

#### 2. Сбор и разметка датасета.

#### Для Wake Word Detection

- Через интерфейс Data Acquisition в Edge Impulse Studio записать с помощью микрофона ПК не менее 50-100 сэмплов с произнесением ключевого слова (класс "wake_word").

- Записать не менее 50-100 сэмплов фонового шума и случайной речи (класс "noise" и "unknown").

- Рекомендация: для улучшения качества модели добавить сэмплы от разных дикторов и в разных акустических условиях.

#### Для классификации жестов

- Использовать мобильное приложение Edge Impulse или встроенный датчик платы для записи сэмплов различных движений (например, "tap", "shake", "wave") — не менее 30-50 сэмплов каждого класса.

- Произвести разметку каждого сэмпла с указанием соответствующей метки класса.

- Разделить датасет на тренировочную (80%) и тестовую (20%) выборки с помощью функции "Auto-split" в Studio.

#### 3. Проектирование импульса (Impulse Design).

- Перейти в раздел "Impulse Design" в Edge Impulse Studio.

#### Добавить блок обработки сигналов (Processing Block)

- Для аудио (Wake Word): добавить блок Audio (MFCC). Это стандартный подход для извлечения признаков из речи: спектрограмма преобразуется в мел-кепстральные коэффициенты, которые подаются на вход нейросети.

- Для акселерометра (жесты): добавить блок Spectral Analysis или Time Series Data, который извлекает статистические признаки из временных рядов.

- Добавить блок обучения (Learning Block): выбрать Classification (Keras) для многоклассовой классификации.

- Настроить параметры окна: для MFCC рекомендуется использовать длину окна 30 мс с перекрытием 20 мс. Количество коэффициентов — 13.

- Выбрать целевую архитектуру: использовать предложенную Edge Impulse архитектуру или настроить вручную (например, сверточная нейронная сеть 1D для аудио или полносвязная для акселерометра).

#### 4. Обучение модели и оценка качества.

- В разделе "Training" запустить обучение модели. Edge Impulse автоматически выполнит оптимизацию гиперпараметров.

- Включить квантование в INT8 (настройка в параметрах обучения) для уменьшения размера модели и ускорения инференса на ESP32-S3.

#### Оценить качество модели

- Просмотреть матрицу ошибок (Confusion Matrix) и точность на тестовой выборке.

- Для Wake Word задач оценить частоту ложных срабатываний (False Positive Rate) и пропусков целевого слова (False Negative Rate).

- Использовать встроенный инструмент Live Classification для тестирования модели в реальном времени через микрофон браузера.

- При необходимости провести дополнительное обучение с увеличенным датасетом или изменением параметров импульса.

#### 5. Развертывание модели на ESP32-S3.

#### В разделе "Deployment" выбрать целевую платформу

- Выбрать "ESP32-S3" или "Arduino" в качестве целевой платформы.

- Включить опции: Quantized (INT8) и EON Compiler (Edge Impulse's proprietary compiler для дополнительной оптимизации).

#### Сгенерировать C++ библиотеку (ZIP-архив), содержащую

- Код модели в виде массива данных (веса и архитектура).

- Интерфейс для выполнения инференса (функции run_classifier и classifier_init).

- Заголовочные файлы с настройками входных/выходных тензоров.

- Распаковать архив и интегрировать его в проект PlatformIO для ESP32-S3.

- Включить в проект необходимые зависимости: библиотеки TensorFlow Lite Micro (если не используется EON) или встроенную в библиотеку Edge Impulse среду выполнения.

#### 6. Реализация инференса на ESP32-S3.

#### Создать основную программу в src/main.cpp

- Инициализировать микрофон (для аудио) через I2S или акселерометр через I2C.

- Инициализировать модель с помощью функции classifier_init().

- Настроить буфер для захвата данных (длиной, соответствующей окну, указанному в Edge Impulse).

#### В основном цикле (loop())

- Захватить блок данных (аудиофрагмент длительностью, например, 1 секунда).

- Заполнить входной тензор модели данными с помощью classifier_set_input() или прямого доступа к буферу.

- Выполнить инференс с помощью classifier_run().

- Получить результаты: массив вероятностей для каждого класса и индекс предсказанного класса.

- Вывести результат в UART и выполнить действие (например, зажечь светодиод при обнаружении Wake Word).

- Для Wake Word задач дополнительно реализовать порог уверенности (confidence threshold) для снижения ложных срабатываний.

#### 7. Измерение и анализ производительности.

#### Замерить ключевые метрики на целевом устройстве

- Время инференса: Использовать esp_timer_get_time() до и после вызова classifier_run(). Вычислить среднее по 100 запускам.

- Использование RAM: В начале программы вывести свободную память (heap) с помощью esp_get_free_heap_size().

- Размер модели: Указать размер массива весов в прошивке (Flash-память).

- Энергопотребление: (Опционально) с помощью USB-амперметра измерить ток в режиме инференса.

- Сравнить результаты с теоретическими ожиданиями (указанными в Edge Impulse Resource Estimator).

- Проверить стабильность работы в реальных условиях (запуск 50+ раз подряд, проверка отсутствия утечек памяти).

## КИМ Р5.2. Object Detection с использованием RKNN-Toolkit2

**Форма проведения:** очная.

**Продолжительность по РПД:** 2 академических часа.

### Цель работы

Практическое освоение полного пайплайна развертывания нейросетевых моделей обнаружения объектов (Object Detection) на платформах с NPU-ускорителем Rockchip с использованием инструментария RKNN-Toolkit2. В рамках работы выполняется конвертация предобученной модели (YOLOv5/YOLOv8/YOLOv11/YOLOv26) в оптимизированный RKNN-формат, квантование для повышения производительности, оценка качества и развертывание на целевом устройстве с выполнением инференса в реальном времени. Особое внимание уделяется анализу влияния квантования на точность и производительность, а также практическим аспектам работы с аппаратным ускорителем NPU.

### Задача работы

Реализовать пайплайн развертывания модели Object Detection на базе семейства YOLO на плате с процессором Rockchip (например, RK3588, RK3568 или аналоги) с использованием RKNN-Toolkit2 :

- Выбрать модель обнаружения объектов (YOLOv5, YOLOv8, YOLOv11, YOLOv26) и получить её в формате ONNX.

- Установить и настроить среду RKNN-Toolkit2 на PC (Ubuntu) .

- Выполнить конвертацию ONNX-модели в RKNN-формат с квантованием INT8 для целевой платформы .

- Оценить потерю точности при квантовании (сравнение FP16 vs INT8).

- Развернуть модель на целевом устройстве и выполнить инференс на тестовых изображениях и с веб-камеры в реальном времени .

- Провести замеры производительности (FPS, latency, использование памяти) и сравнить с теоретическими показателями. Метриками оптимизации будут являться:

- Точность модели на тестовой выборке (mAP) для FP16 и INT8 вариантов.

- Время инференса (латентность) и частота кадров (FPS) на целевом устройстве.

- Размер модели в формате ONNX vs RKNN (коэффициент сжатия).

- Загрузка NPU и использование DRAM во время инференса.

- Качество детекции (визуальное сравнение результатов).

### Необходимое оборудование и программное обеспечение

- Платформа с NPU Rockchip: например, Radxa Rock 5B (RK3588), Orange Pi 5, Luckfox Aura (RV1126) или reComputer RK3588/RK3576 .

- Веб-камера (USB или MIPI-CSI) для тестирования в реальном времени.

- Персональный компьютер с установленной ОС Ubuntu.

#### Программное обеспечение на стороне ПК

- Python , Conda (рекомендуется) .

- RKNN-Toolkit2 (пакет для конвертации и симуляции на PC) .

- ONNX Runtime или PyTorch для верификации исходной модели.

- Утилиты: adb (для отладки по USB), scp (для копирования файлов на устройство).

#### Программное обеспечение на устройстве

- RKNN-Toolkit2-Lite (Python API для выполнения инференса на NPU) .

- Библиотека rknpu2 (драйвер NPU).

- Предобученная модель YOLO в формате ONNX (например, yolov8n.onnx из rknn_model_zoo) .

- Тестовый набор изображений (например, из COCO-датасета) или видеофайл.

### Задания для выполнения

#### 1. Подготовка среды RKNN-Toolkit2 на ПК.

- Установить Ubuntu на ПК (виртуальная машина или WSL2 с поддержкой USB-проброса) .

#### Установить зависимости для RKNN-Toolkit2 

- bash

- sudo apt install git python3-pip libxslt1-dev zlib1g-dev libglib2.0-0 libsm6 libgl1-mesa-glx libprotobuf-dev build-essential adb

#### Создать Conda-окружение (рекомендуется) и установить RKNN-Toolkit2

- bash

- conda create -n rknn python=3.8

- conda activate rknn

- git clone -b v2.3.0 https://github.com/airockchip/rknn-toolkit2.git

- cd rknn-toolkit2/rknn-toolkit2/packages/x86_64/

- pip install -r requirements_cp38-2.3.0.txt

- pip install ./rknn_toolkit2-2.3.0-cp38-cp38-manylinux_2_17_x86_64.manylinux2014_x86_64.whl

- Проверить установку: python -c "from rknn.api import RKNN; print('OK')" .

#### 2. Получение и подготовка модели (YOLO → ONNX).

#### Клонировать репозиторий rknn_model_zoo для доступа к скриптам и примерам 

- bash

- git clone -b v2.3.0 https://github.com/airockchip/rknn_model_zoo.git

- Загрузить предобученную модель YOLOv8n в формате ONNX (или выполнить экспорт из PyTorch/Ultralytics):

- bash

- cd rknn_model_zoo/examples/yolov8/model

- bash download_model.sh

- Верифицировать ONNX-модель: выполнить инференс на ПК с помощью ONNX Runtime, сохранить эталонные результаты (bounding boxes, классы, уверенность).

#### 3. Конвертация модели в RKNN-формат с квантованием INT8.

- Использовать скрипт convert.py из rknn_model_zoo/examples/yolov8/python/ для конвертации ONNX → RKNN :

- bash

- cd rknn_model_zoo/examples/yolov8/python

- python3 convert.py ../model/yolov8n.onnx rk3588 i8 ../model/yolov8n_rk3588_i8.rknn

- Параметр rk3588 — целевая платформа. Для RK3566/RK3568 указать rk3566/rk3568 соответственно .

- Параметр i8 — INT8 квантование (для максимальной производительности). Альтернатива — fp (FP16) для сравнения .

- Выполнить конвертацию также с параметром fp для получения FP16-версии модели.

- Зафиксировать размеры полученных .rknn-файлов и вычислить коэффициенты сжатия относительно исходного ONNX.

#### 4. Анализ качества квантованной модели (на ПК).

- Выполнить симуляцию инференса RKNN-моделей на ПК (без подключения устройства) с использованием API RKNN :

- python

- from rknn.api import RKNN

- rknn = RKNN()

- rknn.load_rknn('yolov8n_rk3588_i8.rknn')

- rknn.init_runtime() # run on simulator

- outputs = rknn.inference(inputs=[input_data])

- Сравнить результаты (bounding boxes, классы) INT8-модели с эталонными результатами ONNX-модели на тестовом наборе из 50-100 изображений.

- Вычислить метрику mAP (mean Average Precision) для FP16 и INT8 вариантов.

- Зафиксировать падение точности после INT8-квантования.

- Визуализировать результаты детекции (наложение bounding boxes на изображения) для визуального сравнения.

#### 5. Развертывание и инференс на целевом устройстве.

- Подключить целевое устройство к ПК по USB (или по сети). Убедиться, что устройство видно через adb devices .

#### Скопировать RKNN-модель на устройство

- bash

- adb push yolov8n_rk3588_i8.rknn /data/

#### На устройстве установить RKNN-Toolkit2-Lite (если не установлен в образе)

- bash

- sudo apt update

- sudo apt install python3-rknnlite2 # для RK3588

- # или

- sudo apt install rknpu2-rk3588 python3-rknnlite2

#### Для RK356X платформ необходимо включить NPU через rsetup 

- bash

- sudo rsetup -> Overlays -> Manage overlays -> Enable NPU

- Выполнить инференс модели на устройстве на тестовых изображениях (например, bus.jpg) с использованием скрипта yolov8.py :

- bash

- cd /usr/share/rknn_model_zoo/examples/yolov8/python

- python3 yolov8.py --model_path /data/yolov8n_rk3588_i8.rknn --img_save

- Результат должен содержать координаты, классы и уверенность для обнаруженных объектов .

#### 6. Замеры производительности и анализ эффективности.

- Измерить время инференса на устройстве: использовать скрипты, которые выводят задержку в миллисекундах.

- Выполнить инференс на 100-200 изображениях, вычислить среднюю латентность, минимальное и максимальное время.

#### Измерить частоту кадров (FPS) при работе с веб-камерой

- Запустить скрипт детекции в реальном времени (например, web_detection.py из репозитория Seeed Studio) .

- Зафиксировать FPS на устройстве.

- Сравнить производительность INT8 и FP16 моделей.

- Оценить использование DRAM и загрузку NPU (из логов RKNN Runtime) .

- Тест производительности: Запуск 100+ инференсов подряд без падения FPS и ошибок памяти.

- Тест на разных разрешениях: Проверка работы модели при различных входных размерах (если модель поддерживает динамическую форму).

- Отсутствие артефактов детекции (ложные срабатывания, искаженные bounding boxes) на тестовом наборе.
