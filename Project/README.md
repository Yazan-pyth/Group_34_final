# Проектная работа

## Назначение

Проектная работа является интегральным контрольно-измерительным материалом дисциплины. Обучающийся должен реализовать работоспособный прототип Edge AI-системы и показать полный путь от подготовки данных и обучения модели до развёртывания на MCU или NPU.

## Варианты отраслевой траектории

### ТЭК

Автономный датчик предиктивной диагностики насоса:

- сбор вибрационных или акустических данных;
- TinyML-модель детекции аномалий на ESP32;
- передача телеметрии по MQTT;
- оптимизация программного энергопотребления для длительной автономной работы.

Проверяемые индикаторы: `PL-4.2`, `DL-1.11`, `LC-5.1`.

### Транспорт

Edge AI-модуль детекции препятствий для беспилотного погрузчика:

- Orange Pi 5 или другая платформа Rockchip RK3588/RK3576;
- объектная детекция с использованием RKNN;
- интеграция с системой реального времени;
- latency менее 50 мс;
- валидация на пользовательском датасете.

Проверяемые индикаторы: `PL-4.2`, `DL-1.11`, `DL-3.3`, `LC-5.1`.

## Обязательные результаты

- исходный код и конфигурации сборки;
- подготовленная и конвертированная модель;
- работающий прототип на целевой платформе;
- результаты измерения качества, latency, throughput, RAM/Flash/DRAM и энергопотребления — в зависимости от выбранной платформы;
- технический отчёт;
- демонстрационные материалы для защиты.

## Критерии оценки

| Критерий | Вес | Содержание |
|---|---:|---|
| Функциональность | 25% | Корректность работы модели на целевом устройстве и выполнение требований ТЗ |
| Оптимизация | 25% | Сжатие модели, размер, latency и точность; обоснование компромисса «размер — точность» |
| Надёжность | 25% | Работа в real-time, обработка прерываний, сбоев и режимов энергосбережения |
| Отраслевая релевантность | 25% | Соответствие условиям ТЭК или Транспорта и корректное обоснование решений |

Защита итогового проекта составляет 50% итоговой оценки по дисциплине. Подробное описание шкалы находится в [методических указаниях по защите](../Exam/independent-work.md).
